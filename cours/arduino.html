ARD.01
Introduction à ArduinoMicrocontrôleur open-source pour l'électronique

Qu'est-ce qu'Arduino ?

Arduino est une plateforme de prototypage électronique open-source basée sur un microcontrôleur. Elle permet de lire des capteurs, contrôler des LED, moteurs, afficheurs, et communiquer avec d'autres appareils.

Arduino UNO
Modèle le plus populaire pour débutants
ATmega328P • 14 GPIO • 6 PWM
Arduino NANO
Version compacte pour projets embarqués
Mini-USB • Breadboard friendly
Arduino MEGA
Plus de mémoire et GPIO (54 pins)
Pour projets complexes

Arduino IDE : Installation

Téléchargez l'Arduino IDE (Integrated Development Environment) depuis arduino.cc/software. C'est l'éditeur officiel pour programmer vos cartes Arduino.

Alternative moderne : Arduino IDE 2.0 offre une interface améliorée avec auto-complétion et débogage.
Premiers pas
  • Arduino se programme en C/C++ (syntaxe simplifiée)
  • Structure de base : setup() (initialisation) et loop() (boucle infinie)
  • Brancher la carte via USB, sélectionner le port COM dans Arduino IDE
  • Cliquer sur "Téléverser" (Upload) pour envoyer le code sur la carte
ARD.02
Premier programme : BlinkFaire clignoter une LED

Code Blink (LED intégrée)

Arduino C++ 
// Fonction exécutée une fois au démarrage
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);  // LED intégrée (pin 13 sur UNO)
}

// Fonction exécutée en boucle infinie
void loop() {
  digitalWrite(LED_BUILTIN, HIGH);  // Allumer
  delay(1000);                      // Attendre 1 seconde
  digitalWrite(LED_BUILTIN, LOW);   // Éteindre
  delay(1000);                      // Attendre 1 seconde
}

Contrôle d'une LED externe

Connectez une LED avec une résistance 220Ω entre la pin 9 et GND.

Arduino C++ 
const int LED_PIN = 9;  // Broche numérique 9

void setup() {
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_PIN, HIGH);
  delay(500);
  digitalWrite(LED_PIN, LOW);
  delay(500);
}

PWM : Variation d'intensité lumineuse

Les broches PWM (~3, ~5, ~6, ~9, ~10, ~11 sur Arduino UNO) permettent de simuler un signal analogique entre 0 (0V) et 255 (5V).

Arduino C++ 
const int LED_PIN = 9;

void setup() {
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  // Augmenter progressivement l'intensité
  for (int i = 0; i <= 255; i++) {
    analogWrite(LED_PIN, i);
    delay(10);
  }
  
  // Diminuer progressivement
  for (int i = 255; i >= 0; i--) {
    analogWrite(LED_PIN, i);
    delay(10);
  }
}
ARD.03
Entrées numériquesLecture de boutons et interrupteurs

Lire un bouton

Arduino C++ 
const int BUTTON_PIN = 2;
const int LED_PIN = 13;

void setup() {
  pinMode(BUTTON_PIN, INPUT_PULLUP);  // Résistance pull-up interne
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  int buttonState = digitalRead(BUTTON_PIN);
  
  if (buttonState == LOW) {  // Bouton appuyé (masse)
    digitalWrite(LED_PIN, HIGH);
  } else {
    digitalWrite(LED_PIN, LOW);
  }
}

Débounce : éviter les rebonds

Les boutons mécaniques génèrent des "rebonds" (plusieurs signaux lors d'un appui). On ajoute un délai :

Arduino C++ 
const int BUTTON_PIN = 2;
int lastButtonState = HIGH;
int ledState = LOW;

void setup() {
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  int buttonState = digitalRead(BUTTON_PIN);
  
  if (buttonState == LOW && lastButtonState == HIGH) {
    ledState = !ledState;  // Inverser l'état
    digitalWrite(LED_BUILTIN, ledState);
    delay(50);  // Debounce de 50ms
  }
  
  lastButtonState = buttonState;
}
ARD.04
Entrées analogiquesPotentiomètres, capteurs de lumière (LDR), température

Lire un potentiomètre

Les broches A0-A5 lisent des valeurs analogiques (0-1023 correspondant à 0-5V).

Arduino C++ 
const int POT_PIN = A0;
const int LED_PIN = 9;

void setup() {
  pinMode(LED_PIN, OUTPUT);
  Serial.begin(9600);  // Communication série (moniteur série)
}

void loop() {
  int potValue = analogRead(POT_PIN);  // 0-1023
  int brightness = map(potValue, 0, 1023, 0, 255);  // Convertir en 0-255
  
  analogWrite(LED_PIN, brightness);
  
  Serial.print("Potentiomètre : ");
  Serial.println(potValue);
  
  delay(100);
}
ESP.01
Introduction à l'ESP32Microcontrôleur avec WiFi et Bluetooth intégrés

Qu'est-ce que l'ESP32 ?

L'ESP32 est un microcontrôleur low-cost développé par Espressif Systems. Il embarque le WiFi et le Bluetooth, ce qui en fait un choix idéal pour les projets IoT.

Caractéristique ESP32 Arduino UNO
Processeur Dual-core 240 MHz Single-core 16 MHz
Mémoire Flash 4 Mo 32 Ko
RAM 520 Ko 2 Ko
WiFi ✓ Intégré ✗ Module externe
Bluetooth ✓ BLE 4.2 ✗ Non
GPIO 34 pins 14 pins
Prix ~3-8€ ~20€

Configuration Arduino IDE pour ESP32

Pour programmer l'ESP32 avec Arduino IDE :

  1. Ouvrir Fichier → Préférences
  2. Dans "URL de gestionnaire de cartes supplémentaires", ajouter :
    https://dl.espressif.com/dl/package_esp32_index.json
  3. Aller dans Outils → Type de carte → Gestionnaire de carte
  4. Chercher "ESP32" et installer le package officiel
  5. Sélectionner votre modèle : ESP32 Dev Module (le plus courant)
ESP.02
ESP32 : Connexion WiFiSe connecter à un réseau et créer un serveur web

Connexion à un réseau WiFi

Arduino C++ (ESP32) 
#include <WiFi.h>

const char* ssid = "VotreSSID";
const char* password = "VotreMotDePasse";

void setup() {
  Serial.begin(115200);
  
  // Connexion WiFi
  WiFi.begin(ssid, password);
  Serial.print("Connexion WiFi");
  
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  
  Serial.println("\nConnecté !");
  Serial.print("Adresse IP : ");
  Serial.println(WiFi.localIP());
}

void loop() {
  // Votre code ici
}

Créer un serveur web simple

Arduino C++ (ESP32) 
#include <WiFi.h>
#include <WebServer.h>

const char* ssid = "VotreSSID";
const char* password = "VotreMotDePasse";

WebServer server(80);

void handleRoot() {
  server.send(200, "text/html", "<h1>Bonjour depuis ESP32 !</h1>");
}

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  
  Serial.println("\nIP : " + WiFi.localIP().toString());
  
  // Routes
  server.on("/", handleRoot);
  server.begin();
  Serial.println("Serveur démarré");
}

void loop() {
  server.handleClient();
}
Test : Une fois le code uploadé, ouvrez un navigateur et tapez l'adresse IP affichée dans le moniteur série (ex: http://192.168.1.50). Vous verrez la page HTML !

Projet 1 : Mesure de température avec DHT11

Arduino C++ 
// Installer la librairie : DHT sensor library by Adafruit
#include <DHT.h>

#define DHTPIN 2
#define DHTTYPE DHT11

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(9600);
  dht.begin();
}

void loop() {
  float temperature = dht.readTemperature();
  float humidite = dht.readHumidity();
  
  if (isnan(temperature) || isnan(humidite)) {
    Serial.println("Erreur de lecture");
    return;
  }
  
  Serial.print("Température : ");
  Serial.print(temperature);
  Serial.println(" °C");
  
  Serial.print("Humidité : ");
  Serial.print(humidite);
  Serial.println(" %");
  
  delay(2000);
}

Exercices & Travaux PratiquesMise en pratique des notions Arduino & ESP32

Cette section regroupe tous les exercices progressifs et travaux pratiques qui accompagnent le cours Arduino & ESP32. Les exercices sont classés par niveau de difficulté.

💡
Avant de commencer : Assurez-vous d'avoir installé l'Arduino IDE, d'avoir branché votre carte via USB, et d'avoir sélectionné le bon port COM dans Outils > Port.

Exercices de compréhension

📝 Exercice 1 — Le clignotement paramétré Facile

  1. Modifiez le programme Blink de base pour que la LED clignote 5 fois rapidement (100 ms), puis 2 fois lentement (1 seconde), et ainsi de suite en boucle.
  2. Utilisez des constantes (const int) pour définir la durée courte et la durée longue.
  3. Ajoutez un message sur le moniteur série indiquant l'état de la LED ("LED ON" / "LED OFF").
  4. Bonus : Utilisez un tableau int delays[] = {100,100,100,100,100,1000,1000}; pour piloter le rythme en parcourant le tableau avec une boucle for.

📝 Exercice 2 — Feu tricolore Moyen

  1. Connectez 3 LEDs (rouge, orange, verte) aux broches 9, 10, 11.
  2. Programmez un feu tricolore : vert 3 s → orange 1 s → rouge 3 s → répétition.
  3. Ajoutez un mode nuit : quand un bouton (broche 2) est maintenu appuyé, seule la LED orange clignote à 0,5 Hz.
  4. Question : Pourquoi utilise-t-on des résistances de 220 Ω en série avec les LEDs ? Calculez la valeur exacte pour une LED rouge (Vf = 2V, If = 20mA, Vcc = 5V).

📝 Exercice 3 — Capteur de lumière réactif Moyen

  1. Branchez une photorésistance (LDR) en diviseur de tension sur la broche A0.
  2. Lisez la valeur analogique et affichez-la toutes les 500 ms sur le moniteur série.
  3. Allumez une LED si la valeur est inférieure à 300 (obscurité), éteignez-la sinon.
  4. Utilisez map() pour convertir la plage 0-1023 vers une luminosité PWM 0-255 et contrôler l'intensité d'une LED via analogWrite().
  5. Réflexion : Quelle est la résolution du convertisseur ADC de l'Arduino Uno ? Quelle tension correspond à la valeur 512 ?

📝 Exercice 4 — ESP32 WiFi + LED Web Avancé

  1. Créez un serveur Web minimal sur l'ESP32 qui affiche une page HTML avec deux boutons : "Allumer" et "Éteindre".
  2. Chaque bouton envoie une requête GET /on ou /off qui contrôle la LED GPIO2.
  3. Ajoutez sur la page l'état actuel de la LED en temps réel (rechargement automatique toutes les 2 s avec meta refresh).
  4. Bonus : Ajoutez un slider HTML pour régler l'intensité via PWM (requête /pwm?val=X).

Travaux Pratiques

🔬 TP 1 — Station météo Arduino Moyen — 2h

Objectif : Construire une mini-station météo affichant température et humidité sur un écran LCD I2C.

Matériel nécessaire :

  • Arduino Uno ou Nano
  • Capteur DHT11 ou DHT22
  • Écran LCD 16×2 avec module I2C (adresse 0x27)
  • Câbles Dupont, breadboard

Étapes :

  1. Câblage DHT11 : VCC→5V, GND→GND, DATA→broche 7. Ajouter résistance pull-up 10 kΩ entre DATA et VCC.
  2. Câblage LCD I2C : VCC→5V, GND→GND, SDA→A4, SCL→A5.
  3. Bibliothèques : Installer DHT sensor library (Adafruit) et LiquidCrystal_I2C via le gestionnaire de bibliothèques.
  4. Code : Lire la température et l'humidité, les afficher sur les deux lignes du LCD. Mettre à jour toutes les 2 s.
  5. Validation : Comparer les valeurs avec un thermomètre de référence. Calculer l'erreur en %.
  6. Extension : Ajouter une LED rouge qui s'allume si T > 30 °C ou humidité > 80 %. Ajouter un buzzer qui sonne 3 fois en cas d'alerte.
✅
Rendu attendu : Code Arduino commenté + schéma de câblage (Fritzing ou dessin annoté) + tableau de mesures comparatives + rapport de 1 page décrivant les difficultés rencontrées.

🔬 TP 2 — Contrôleur IoT ESP32 avec tableau de bord Web Avancé — 3h

Objectif : Créer un tableau de bord web hébergé sur l'ESP32 pour surveiller et contrôler plusieurs périphériques.

  1. Capteurs : Connecter un DHT22 (T°/humidité), un LDR (luminosité) et un HC-SR04 (distance ultrasonique).
  2. Serveur Web : Utiliser la bibliothèque AsyncWebServer pour servir une page HTML responsive.
  3. API JSON : Créer une route /api/sensors qui retourne les données en JSON (mise à jour via fetch JavaScript toutes les 3 s).
  4. Contrôles : Boutons web pour allumer/éteindre une LED, un relais et un ventilateur DC.
  5. Historique : Stocker les 20 dernières mesures en RAM (tableau circulaire) et les afficher sous forme de graphique ASCII sur la page.
  6. OTA : Activer la mise à jour Over-The-Air (ArduinoOTA) pour flasher le code sans câble USB.
