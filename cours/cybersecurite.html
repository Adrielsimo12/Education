CYB.01
Introduction à la CybersécuritéEnjeux et importance de la sécurité informatique

Qu'est-ce que la cybersécurité ?

La cybersécurité désigne l'ensemble des moyens techniques, organisationnels, juridiques et humains nécessaires pour protéger les systèmes informatiques, les réseaux et les données contre les cyberattaques.

Chiffres clés 2024 : Plus de 4 milliards de cyberattaques recensées dans le monde. Le coût moyen d'une violation de données pour une entreprise : 4,45 millions de dollars.

Les 3 piliers de la sécurité (Triade CIA)

Confidentialité
Les données ne sont accessibles qu'aux personnes autorisées
Chiffrement, contrôle d'accès
Intégrité
Les données ne sont pas modifiées de manière non autorisée
Hash, signature numérique
Disponibilité
Les systèmes et données sont accessibles quand nécessaire
Redondance, sauvegardes
Objectifs de la cybersécurité
  • Protéger les données sensibles contre le vol et la fuite
  • Empêcher les accès non autorisés aux systèmes
  • Détecter et répondre rapidement aux incidents
  • Assurer la continuité des services informatiques
  • Se conformer aux réglementations (RGPD, etc.)

CYB.03
CryptographieChiffrement symétrique, asymétrique et hachage

Qu'est-ce que le chiffrement ?

Le chiffrement est une technique qui transforme des données lisibles (texte clair) en données illisibles (texte chiffré) à l'aide d'une clé. Seule la personne possédant la clé de déchiffrement peut lire le message original.

Chiffrement Symétrique vs Asymétrique

Critère Chiffrement Symétrique Chiffrement Asymétrique
Nombre de clés 1 clé (partagée) 2 clés (publique + privée)
Vitesse Très rapide Plus lent
Sécurité Dépend du partage sécurisé de la clé Très sécurisé
Usage typique Chiffrement de fichiers, disques HTTPS, email chiffré, signatures
Algorithmes AES, DES, 3DES RSA, ECC

Exemple : Chiffrement AES en Python

Python 
from cryptography.fernet import Fernet

# Générer une clé de chiffrement
cle = Fernet.generate_key()
print(f"Clé : {cle}")

# Créer un objet de chiffrement
cipher = Fernet(cle)

# Message à chiffrer
message = b"Ceci est un message secret"

# Chiffrer le message
message_chiffre = cipher.encrypt(message)
print(f"Message chiffré : {message_chiffre}")

# Déchiffrer le message
message_dechiffre = cipher.decrypt(message_chiffre)
print(f"Message déchiffré : {message_dechiffre.decode()}")

# Résultat :
# Message chiffré : gAAAAABk...
# Message déchiffré : Ceci est un message secret

Fonction de Hachage

Le hachage est une fonction à sens unique qui transforme des données de taille variable en une empreinte de taille fixe. Utilisé pour vérifier l'intégrité des données et stocker les mots de passe.

Python (SHA-256) 
import hashlib

# Hacher un mot de passe
mot_de_passe = "MonMotDePasse123"
hash_objet = hashlib.sha256(mot_de_passe.encode())
empreinte = hash_objet.hexdigest()

print(f"Hash SHA-256 : {empreinte}")
# Résultat : a1b2c3d4e5f6... (toujours le même pour le même mot de passe)

# Vérification de l'intégrité
fichier_hash = hashlib.sha256(open('fichier.txt', 'rb').read()).hexdigest()

Sécurité RéseauPare-feu, VPN, protocoles sécurisés

Éléments de sécurité réseau

Pare-feu (Firewall)
Filtre le trafic entrant et sortant
Bloque ports, IPs suspectes
VPN
Tunnel chiffré pour connexions à distance
OpenVPN, WireGuard
IDS/IPS
Détection et prévention d'intrusions
Snort, Suricata
HTTPS
Communication web chiffrée (TLS/SSL)
Certificat SSL, Let's Encrypt

Règles de pare-feu (iptables Linux)

Bash 
# Bloquer tout le trafic entrant par défaut
sudo iptables -P INPUT DROP

# Autoriser le trafic SSH (port 22)
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Autoriser HTTP (80) et HTTPS (443)
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Bloquer une IP spécifique
sudo iptables -A INPUT -s 192.168.1.100 -j DROP

# Voir les règles actives
sudo iptables -L -v
CYB.05
Gestion des Mots de PasseBonnes pratiques et authentification forte

Règles pour un mot de passe fort

Caractéristiques d'un bon mot de passe
  • Au moins 12 caractères
  • Mélange de majuscules, minuscules, chiffres et symboles
  • Pas de mots du dictionnaire ni d'informations personnelles
  • Unique pour chaque service (ne jamais réutiliser)
  • Utiliser un gestionnaire de mots de passe (Bitwarden, KeePass)
❌ Mauvais exemples
password123
MonNom1990
azerty
admin
12345678
✅ Bons exemples
k9#Tz@7mP$qL2&vN
Correct-Horse-Battery-Staple-42!
J'adore_L3s_Cours-CIEL2024!
Xy$9pL@mK#2rT&vQ

Authentification à Deux Facteurs (2FA)

La 2FA ajoute une couche de sécurité supplémentaire en exigeant deux preuves d'identité :

  1. Quelque chose que vous savez : mot de passe
  2. Quelque chose que vous possédez : smartphone, clé USB de sécurité
  3. Quelque chose que vous êtes : empreinte digitale, reconnaissance faciale
Applications 2FA recommandées : Google Authenticator, Microsoft Authenticator, Authy

Sécurité des Applications WebXSS, CSRF, injection SQL et protection

Vulnérabilités web courantes (OWASP Top 10)

Vulnérabilité Description Protection
Injection SQL Insertion de code SQL malveillant Requêtes préparées, validation des entrées
XSS (Cross-Site Scripting) Injection de JavaScript dans une page Échappement des caractères, CSP
CSRF Action non voulue effectuée par un utilisateur authentifié Jetons CSRF, SameSite cookies
Broken Authentication Gestion incorrecte des sessions 2FA, expiration de session

Exemple : Protection contre l'injection SQL

❌ Code vulnérable
# Python Flask - VULNÉRABLE
@app.route('/login')
def login():
    username = request.args.get('user')
    query = f"SELECT * FROM users WHERE username='{username}'"
    # Injection possible : user=' OR '1'='1
    result = db.execute(query)
✅ Code sécurisé
# Python Flask - SÉCURISÉ
@app.route('/login')
def login():
    username = request.args.get('user')
    # Requête préparée (paramétrisée)
    query = "SELECT * FROM users WHERE username=?"
    result = db.execute(query, (username,))

Gestion des IncidentsDétection, réponse et récupération

Cycle de gestion d'incident

1. Détection
Identifier l'incident de sécurité
Logs, alertes IDS
2. Confinement
Isoler le système compromis
Déconnexion réseau
3. Éradication
Supprimer la menace
Suppression malware
4. Récupération
Restaurer les systèmes
Sauvegardes, patches

Actions immédiates en cas d'incident

Checklist de réponse
  • ✓ Alerter le responsable sécurité / DSI
  • ✓ Documenter l'incident (date, heure, symptômes)
  • ✓ Isoler le système affecté du réseau
  • ✓ Ne PAS éteindre brutalement (perte de preuves en RAM)
  • ✓ Capturer les logs et l'état du système
  • ✓ Changer tous les mots de passe potentiellement compromis
  • ✓ Analyser l'étendue de la compromission
CYB.08
Bonnes PratiquesHygiène numérique au quotidien

10 règles d'or de la cybersécurité

À appliquer au quotidien
  • 1. Mettre à jour régulièrement tous les logiciels et le système d'exploitation
  • 2. Utiliser un antivirus à jour et activer le pare-feu
  • 3. Sauvegarder les données importantes (règle 3-2-1)
  • 4. Vérifier les URLs avant de cliquer (survoler avec la souris)
  • 5. Ne jamais partager de mots de passe ou informations sensibles
  • 6. Activer la 2FA sur tous les comptes importants
  • 7. Utiliser HTTPS uniquement pour les sites sensibles
  • 8. Méfiez-vous des clés USB inconnues (vecteur de malware)
  • 9. Verrouiller votre session quand vous vous absentez
  • 10. Former et sensibiliser régulièrement (maillon faible = humain)

TP 2 : Configuration d'un Pare-feu Simple

Objectif : Configurer des règles de pare-feu sur Linux avec iptables

Énoncé : Configurez un pare-feu qui :
  • Bloque tout trafic entrant par défaut
  • Autorise SSH (port 22) uniquement depuis 192.168.1.0/24
  • Autorise HTTP (80) et HTTPS (443) pour tous
  • Bloque l'IP 203.0.113.50 (attaquant connu)
Solution Bash 
#!/bin/bash
# Script de configuration pare-feu

# 1. Politique par défaut : DROP
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# 2. Autoriser le loopback (localhost)
sudo iptables -A INPUT -i lo -j ACCEPT

# 3. Autoriser SSH uniquement depuis 192.168.1.0/24
sudo iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 22 -j ACCEPT

# 4. Autoriser HTTP et HTTPS pour tous
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 5. Bloquer l'IP malveillante
sudo iptables -A INPUT -s 203.0.113.50 -j DROP

# 6. Autoriser les connexions déjà établies
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Sauvegarder les règles
sudo iptables-save > /etc/iptables/rules.v4

echo "Pare-feu configuré avec succès"

TP 3 : Chiffrement de Fichiers Sensibles

Objectif : Chiffrer et déchiffrer des fichiers avec GPG

Énoncé : Vous devez envoyer un fichier confidentiel par email. Utilisez GPG pour le chiffrer de manière sécurisée.
Bash 
# Installation de GPG (si nécessaire)
sudo apt install gnupg

# 1. Chiffrer un fichier avec mot de passe (chiffrement symétrique)
gpg -c fichier_secret.pdf
# Entrer un mot de passe fort
# Crée : fichier_secret.pdf.gpg (fichier chiffré)

# 2. Déchiffrer le fichier
gpg fichier_secret.pdf.gpg
# Entrer le mot de passe
# Restaure : fichier_secret.pdf

# 3. Générer une paire de clés (asymétrique)
gpg --full-generate-key
# Choisir : RSA, 4096 bits, validité 1 an

# 4. Chiffrer pour un destinataire spécifique
gpg --encrypt --recipient destinataire@example.com fichier.txt

# 5. Vérifier le chiffrement
file fichier_secret.pdf.gpg
# Résultat : GPG encrypted data
Conseil : Pour transmettre le mot de passe de déchiffrement au destinataire, utilisez un canal différent (SMS, appel téléphonique) jamais le même email.

Exercices & Travaux PratiquesAnalyse de vulnérabilités, cryptographie et sécurisation des systèmes

La cybersécurité s'apprend par la pratique dans un cadre éthique et légal. Ces exercices utilisent des environnements isolés (machines virtuelles, labs dédiés) pour comprendre les attaques et mettre en œuvre les défenses.

⚠️
Éthique et légalité : Ces exercices doivent UNIQUEMENT être réalisés sur des systèmes vous appartenant ou sur des environnements de lab dédiés (TryHackMe, HackTheBox). Toute attaque sans autorisation est un délit pénal (Art. 323-1 du Code pénal français).

Exercices de compréhension

📝 Exercice 1 — Cryptographie appliquée Facile

  1. En Python, hashé la chaîne "Adriel TATCHUM SIMO" avec MD5, SHA-1, SHA-256. Observez les longueurs et les sorties. Pourquoi MD5 est-il obsolète pour les mots de passe ?
  2. Implémentez un chiffrement César en Python : encrypt("Hello", 13) → ROT13. Créez aussi la fonction de déchiffrement.
  3. Utilisez la bibliothèque cryptography Python pour : générer une paire de clés RSA 2048 bits, chiffrer un message avec la clé publique, déchiffrer avec la clé privée.
  4. Réflexion : Expliquez la différence entre chiffrement symétrique et asymétrique. Donnez des exemples d'algorithmes pour chacun.

📝 Exercice 2 — Analyse de vulnérabilités web (OWASP) Moyen

  1. Sur l'application de lab DVWA installée localement, testez et exploitez (en mode "Low" security) : une injection SQL basique, une XSS réfléchie, une vulnérabilité d'upload de fichier.
  2. Pour chaque vulnérabilité : décrivez le mécanisme d'attaque, montrez le payload utilisé, et proposez la correction de code sécurisée.
  3. Passez en mode "High" security et tentez les mêmes attaques. Analysez les protections mises en place.

📝 Exercice 3 — Sécurisation d'un serveur Linux Avancé

  1. Sur une VM Ubuntu fraîchement installée, appliquez le hardening complet : désactiver root SSH, changer le port SSH (ex: 2222), activer l'authentification par clé uniquement, configurer fail2ban, installer et configurer ufw.
  2. Auditez le serveur avec lynis audit system et corrigez les 10 points d'amélioration les plus critiques.
  3. Configurez un certificat TLS auto-signé pour Apache avec OpenSSL. Comprenez le processus CSR → certificat → configuration Apache.
