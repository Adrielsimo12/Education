Variables et types

Python — Variables 
# Python — pas de déclaration de type, pas de ";"
nom    = "Alice"          # str
age    = 17               # int
note   = 15.5             # float
actif  = True             # bool

# f-string (équivalent des template literals JS)
print(f"Bonjour {nom}, tu as {age} ans")

# Types de collections
liste      = [1, 2, 3, 4]          # list — modifiable
tuple_     = (1, 2, 3)              # tuple — immuable
dico       = {"nom": "Alice", "age": 17# dict
ensemble   = {1, 2, 3# set — valeurs uniques

Conditions et boucles

Python — Conditions et boucles 
# if / elif / else (indentation obligatoire !)
if age >= 18:
    print("Majeur")
elif age >= 16:
    print("Presque majeur")
else:
    print("Mineur")

# for sur une liste
notes = [12, 15, 8, 18]
for note in notes:
    print(f"Note : {note}")

# range() — équivalent de for(i=0; i<5; i++)
for i in range(5):
    print(i)      # 0, 1, 2, 3, 4

# Compréhension de liste (puissant et pythonique)
doubles = [n * 2 for n in notes if n >= 10]
L'indentation (4 espaces) remplace les accolades en Python — c'est obligatoire ! Le code indenté forme un bloc.

Fonctions

Python — Fonctions 
# Fonction simple
def saluer(prenom, titre="M."):
    return f"Bonjour {titre} {prenom}"

print(saluer("Dupont"))           # "Bonjour M. Dupont"
print(saluer("Martin", "Mme"))    # "Bonjour Mme Martin"

# Fonction avec plusieurs valeurs retournées
def stats(liste):
    return min(liste), max(liste), sum(liste)/len(liste)

mini, maxi, moy = stats([8, 12, 15, 18])

# Dictionnaires — très utilisés avec Flask/JSON

À retenir — Python
  • L'indentation (4 espaces) remplace les accolades
  • Pas de ; en fin de ligne, pas de var/let/const
  • Les f-strings : f"Bonjour {variable}"
  • Les dictionnaires Python = les objets JavaScript
PY.02
Environnement virtuel (.venv)Isoler les dépendances Python

Créer et activer un environnement

Un environnement virtuel isole les bibliothèques Python du projet. C'est la bonne pratique systématique avant tout projet Python.

CMD / PowerShell (Windows) 
# 1. Créer le dossier projet
mkdir mon_projet
cd mon_projet

# 2. Créer l'environnement virtuel
python -m venv .venv

# 3. Activer l'environnement (PowerShell Windows)
.\.venv\Scripts\Activate.ps1

# 3b. Si erreur de politique d'exécution :
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

# 4. Installer Flask et les dépendances
pip install flask flask-cors pymysql

# 5. Sauvegarder les dépendances
pip freeze > requirements.txt

# 6. Réinstaller depuis requirements.txt
pip install -r requirements.txt

# Désactiver l'environnement
deactivate
Important : toujours voir (.venv) devant votre invite de commande avant de lancer Flask. Sinon les bibliothèques ne seront pas trouvées.
PY.03
Flask — API RESTRoutes, JSON, méthodes HTTP

Premier serveur Flask

Python — app.py 
from flask import Flask, jsonify, request
from flask_cors import CORS

app = Flask(__name__)
CORS(app)  # Autorise les requêtes cross-origin (depuis le front)

# ── Route GET simple ──────────────────────────
@app.route('/api/bonjour')
def bonjour():
    return jsonify({"message": "Bonjour depuis Flask !"})

# ── Route avec paramètre d'URL ────────────────
@app.route('/api/utilisateur/<int:id>')
def get_user(id):
    return jsonify({"id": id, "nom": "Alice"})

# ── Route POST — recevoir des données JSON ────
@app.route('/api/utilisateur', methods=['POST'])
def create_user():
    data = request.get_json()     # lire le corps JSON
    nom  = data.get('nom', '')
    if not nom:
        return jsonify({"erreur": "nom requis"}), 400
    return jsonify({"message": f"Utilisateur {nom} créé"}), 201

if __name__ == '__main__':
    # host='0.0.0.0' = accessible sur le réseau local !
    app.run(debug=True, host='0.0.0.0', port=5000)

Les méthodes HTTP

Méthode Usage Exemple
GET Lire / récupérer des données GET /api/produits
POST Créer une ressource POST /api/produits
PUT Modifier complètement PUT /api/produits/5
PATCH Modifier partiellement PATCH /api/produits/5
DELETE Supprimer DELETE /api/produits/5

Codes HTTP courants

Code Signification Usage
200 OK Requête réussie
201 Created Ressource créée (POST)
400 Bad Request Données invalides
401 Unauthorized Non authentifié
404 Not Found Ressource introuvable
500 Internal Server Error Erreur serveur
À retenir — Flask
  • Toujours activer .venv avant de travailler
  • @app.route('/chemin') définit une URL de l'API
  • jsonify() convertit un dict Python en JSON HTTP
  • host='0.0.0.0' rend l'API accessible sur le réseau local

Exercices & Travaux PratiquesPython algorithmique, Flask web et API REST

Ces exercices progressifs vous permettront de maîtriser Python et le framework Flask, depuis les bases du langage jusqu'à la création d'API REST complètes.

🐍
Environnement : Travaillez toujours dans un environnement virtuel (python -m venv .venv). Activez-le avant d'installer les dépendances : source .venv/bin/activate (Linux/Mac) ou .venv\Scripts\activate (Windows).

Exercices de compréhension

📝 Exercice 1 — Structures Python fondamentales Facile

  1. Créez un programme qui lit un fichier texte (notes.txt) contenant des notes séparées par des sauts de ligne, calcule la moyenne, la note min et la note max, puis écrit les résultats dans rapport.txt.
  2. Réécrivez le programme précédent en utilisant une list comprehension pour convertir les lignes en nombres.
  3. Utilisez un dictionnaire pour compter la fréquence de chaque mot dans un texte donné (sans utiliser Counter). Affichez les 10 mots les plus fréquents.
  4. Créez un décorateur @timer qui mesure et affiche le temps d'exécution de n'importe quelle fonction.

📝 Exercice 2 — Flask : routes et templates Moyen

  1. Créez une application Flask avec 4 routes : / (accueil), /about, /contact, /articles/<int:id> (article dynamique).
  2. Utilisez Jinja2 pour créer un template base.html avec header/nav/footer, et les autres templates héritent avec {% extends %}.
  3. Passez des données Python (listes, dictionnaires) aux templates et affichez-les avec {% for %} et {% if %}.
  4. Ajoutez des filtres Jinja2 personnalisés : | truncate(100), | date_format.

📝 Exercice 3 — Formulaires Flask & Validation Moyen

  1. Créez un formulaire d'inscription avec Flask-WTF : champs nom, email, mot de passe (avec confirmation), validation côté serveur.
  2. Utilisez flash() pour afficher les messages de succès et d'erreur.
  3. Gérez l'upload de fichier (photo de profil) : restriction sur l'extension (jpg, png, gif), renommage sécurisé avec secure_filename, sauvegarde dans un dossier uploads/.
  4. Implémentez la protection CSRF (Cross-Site Request Forgery) avec le token WTForms.

📝 Exercice 4 — API REST Flask + JSON Avancé

  1. Créez une API REST complète pour gérer une liste de livres : GET /api/books (liste), GET /api/books/<id> (détail), POST /api/books (créer), PUT /api/books/<id> (modifier), DELETE /api/books/<id> (supprimer).
  2. Stockez les données en JSON dans un fichier data.json (pas de base de données).
  3. Ajoutez l'authentification par token API dans le header Authorization: Bearer TOKEN.
  4. Documentez l'API avec des docstrings et générez la documentation avec Flask-RESTX ou Swagger.

Travaux Pratiques

🔬 TP 1 — Blog dynamique avec Flask + SQLite Moyen — 3h

Objectif : Créer un blog fonctionnel avec Flask, SQLite via SQLAlchemy, et authentification.

  1. Modèles SQLAlchemy : User (id, email, password_hash, created_at) et Post (id, title, slug, content, published, user_id, created_at).
  2. Authentification : Inscription, connexion, déconnexion avec Flask-Login. Hash des mots de passe avec werkzeug.security.
  3. CRUD articles : Créer, lire, modifier, supprimer des articles. Seul l'auteur peut modifier/supprimer ses articles.
  4. Slugs : Générer automatiquement des slugs d'URL depuis le titre (ex: "Mon article" → "mon-article").
  5. Pagination : Afficher les articles par pages de 10 avec Flask-SQLAlchemy .paginate().
  6. Rendu : Application déployée localement + code sur GitHub avec README d'installation.

🔬 TP 2 — Scraper web & Tableau de bord Avancé — 3h

Objectif : Collecter des données web automatiquement et les visualiser dans un dashboard Flask.

  1. Scraping : Utiliser requests + BeautifulSoup4 pour scraper les titres et prix d'une boutique demo (books.toscrape.com).
  2. Stockage : Sauvegarder les données dans une base SQLite. Ajouter un timestamp et éviter les doublons.
  3. Dashboard Flask : Page principale avec statistiques (nombre de livres, prix moyen, min, max par catégorie).
  4. Graphiques : Intégrer Chart.js via l'API JSON Flask pour afficher la distribution des prix.
  5. Automatisation : Lancer le scraping périodiquement avec APScheduler. Afficher la date du dernier scraping.
