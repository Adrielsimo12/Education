JS.01
Variables & Types de donnéeslet, const, var — string, number, boolean, object

Déclarer une variable

Une variable est une boîte nommée qui stocke une valeur. En JavaScript moderne, on utilise let ou const.

Mot-clé Usage Modifiable ?
let Variable dont la valeur peut changer ✓ Oui
const Constante — valeur fixe ✗ Non
var Ancienne syntaxe (à éviter) Héritage
JavaScript — Variables 
let nom     = 'Alice';          // chaîne de caractères
let age     = 17;               // nombre entier
let moyenne = 13.5;             // nombre décimal
let majeur  = false;           // booléen
const PI    = 3.14159;         // constante

nom = 'Bob';                    // ✓ let : on peut modifier
// PI = 3;                      ✗ const : TypeError !

Les types de données

string
Texte entre guillemets
"Bonjour" 'JS' `${nom}`
number
Entier ou décimal
42 3.14 -7
boolean
Vrai ou faux
true false
undefined
Non initialisé
let x; // undefined
null
Absence volontaire
let d = null;
object
Structure complexe
{nom: 'Alice'} [1,2,3]

Les template literals

JavaScript — Template literals 
let prenom = 'Marie';
let note   = 16;

// ❌ Concaténation classique (vieille méthode)
let msg1 = 'Bonjour ' + prenom + ', ta note est ' + note;

// ✅ Template literal (méthode moderne)
let msg2 = `Bonjour ${prenom}, ta note est ${note}/20`;

console.log(msg2); // Bonjour Marie, ta note est 16/20

JS.02
Structures conditionnellesif / else — switch — opérateur ternaire

if / else if / else

JavaScript — Conditions 
let note = 14;

if (note >= 16) {
  console.log('Très bien ✨');
} else if (note >= 14) {
  console.log('Bien 👍');       // ← ce bloc s'exécute
} else if (note >= 12) {
  console.log('Assez bien');
} else if (note >= 10) {
  console.log('Passable');
} else {
  console.log('Insuffisant');
}

Opérateur ternaire

Version compacte du if/else sur une ligne : condition ? siVrai : siFaux

JavaScript — Ternaire 
let age    = 17;
let statut = age >= 18 ? 'Majeur' : 'Mineur';
console.log(statut); // 'Mineur'

Opérateurs de comparaison

Toujours utiliser === (égalité stricte) plutôt que == (égalité faible). 5 == '5' vaut true à cause de la conversion de type — c'est une source de bugs !
✅ Correct
if (age === 18) {
  // comparaison stricte
  // type ET valeur
}
❌ À éviter
if (age == '18') {
  // '18' == 18 → true !
  // comportement surprenant
}
JS.03
Bouclesfor — while — forEach — for…of

La boucle for

JavaScript — Boucle for 
// for (initialisation ; condition ; mise à jour)
for (let i = 0; i < 5; i++) {
  console.log(`Tour n°${i}`);
}
// Affiche : Tour n°0, Tour n°1, ..., Tour n°4

forEach et for…of

JavaScript — Parcourir un tableau 
let fruits = ['pomme', 'banane', 'cerise'];

// for...of : syntaxe simple
for (let fruit of fruits) {
  console.log(fruit);
}

// forEach : avec index
fruits.forEach(function(fruit, index) {
  console.log(`${index + 1}. ${fruit}`);
});

// Version fléchée (plus concise)
fruits.forEach((f, i) => console.log(`${i}: ${f}`));

JS.04
FonctionsDéclaration, expression, flèche, portée

Les 3 façons d'écrire une fonction

JavaScript — Fonctions 
// 1. Déclaration (function declaration)
function additionner(a, b) {
  return a + b;
}

// 2. Expression (function expression)
const multiplier = function(a, b) {
  return a * b;
};

// 3. Fonction fléchée (arrow function) — moderne
const diviser = (a, b) => a / b;

// Avec paramètre par défaut
const saluer = (nom, lang = 'fr') => {
  return lang === 'fr' ? `Bonjour ${nom}` : `Hello ${nom}`;
};

JS.05
Tableaux (Array)Créer, parcourir, transformer

Méthodes de base

JavaScript — Tableaux 
let fruits = ['pomme', 'banane', 'cerise'];

fruits[0];              // 'pomme' — indices commencent à 0
fruits.length;          // 3
fruits[fruits.length - 1]; // 'cerise' — dernier élément

fruits.push('kiwi');    // ajouter à la fin
fruits.pop();            // retirer le dernier
fruits.unshift('ananas'); // ajouter au début
fruits.shift();          // retirer le premier
fruits.includes('banane'); // true
fruits.indexOf('cerise');  // 1

map, filter, reduce

JavaScript — Méthodes avancées 
let notes = [8, 15, 12, 18, 10, 14];

// map : transforme chaque élément → nouveau tableau
let doubles = notes.map(n => n * 2);
// [16, 30, 24, 36, 20, 28]

// filter : garde seulement les éléments qui passent le test
let reçus = notes.filter(n => n >= 10);
// [15, 12, 18, 10, 14]

// reduce : calcule une valeur unique
let somme = notes.reduce((acc, n) => acc + n, 0);
let moyenne = somme / notes.length; // 12.83

// find : premier élément qui satisfait
let premiereSup15 = notes.find(n => n > 15); // 18

JS.07
Les événementsaddEventListener, event, délégation

Écouter un événement

JavaScript — Événements 
const btn = document.getElementById('monBouton');

// Méthode recommandée
btn.addEventListener('click', function(event) {
  console.log('Cliqué !');
  console.log(event.target); // l'élément cliqué
});

// Événements clavier
document.addEventListener('keydown', (e) => {
  console.log(e.key);     // 'Enter', 'a', 'Escape'…
  console.log(e.ctrlKey); // true si Ctrl est enfoncé
});

JS.08
Fetch APIRequêtes HTTP asynchrones

Appeler une API

JavaScript — Fetch 
// GET — récupérer des données
async function chargerUtilisateurs() {
  const reponse = await fetch('http://192.168.1.10:5000/api/utilisateurs');
  const data    = await reponse.json();
  console.log(data);
}

// POST — envoyer des données
async function creerUtilisateur(nom, email) {
  const reponse = await fetch('http://192.168.1.10:5000/api/utilisateur', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ nom, email })
  });
  const resultat = await reponse.json();
  console.log(resultat);
}
À retenir — JavaScript
  • let et const pour déclarer des variables
  • === pour comparer (égalité stricte)
  • map, filter, reduce pour transformer les tableaux
  • querySelector pour sélectionner des éléments DOM
  • addEventListener pour réagir aux interactions
  • fetch avec async/await pour les appels API

Exercices & Travaux PratiquesAlgorithmes, manipulation du DOM, fetch, et projets complets en JavaScript

JavaScript est le langage qui donne vie aux pages web. Ces exercices progressifs vous feront maîtriser les bases algorithmiques, la manipulation du DOM, les appels API asynchrones et la programmation orientée objet.

⚠️
Bonne pratique : Ouvrez toujours la console navigateur (F12) pendant vos exercices. Utilisez console.log() pour déboguer, console.table() pour les tableaux et console.error() pour les erreurs.

Exercices de compréhension

📝 Exercice 1 — Algorithmique de base Facile

  1. Écrivez une fonction isPrime(n) qui retourne true si n est premier, false sinon. Affichez tous les nombres premiers jusqu'à 100.
  2. Implémentez un tri à bulles (bubble sort) sur un tableau de nombres. Comptez le nombre d'opérations effectuées.
  3. Créez une fonction fibonacci(n) en version récursive ET en version itérative. Comparez les performances avec console.time() pour n=40.
  4. Écrivez flatten(arr) qui aplatit un tableau imbriqué à profondeur arbitraire sans utiliser Array.flat().

📝 Exercice 2 — Manipulation du DOM Moyen

  1. Créez dynamiquement une liste de tâches (To-Do) : champ de saisie + bouton "Ajouter" → crée un <li> avec un bouton "Supprimer" et une case à cocher "Terminé".
  2. Sauvegardez et restaurez la liste via localStorage (persistance après rechargement).
  3. Ajoutez un filtre : "Toutes", "En cours", "Terminées" qui masque/affiche les tâches sans supprimer le DOM.
  4. Implémentez le glisser-déposer (drag & drop HTML5) pour réordonner les tâches.

📝 Exercice 3 — Asynchrone & Fetch API Moyen

  1. Appelez l'API publique https://jsonplaceholder.typicode.com/users avec fetch() et affichez les utilisateurs dans des cartes HTML.
  2. Gérez les états : loading (spinner), success (affichage des données), error (message d'erreur en rouge).
  3. Ajoutez une barre de recherche qui filtre les utilisateurs côté client (input event + Array.filter()).
  4. Refactorisez avec async/await et expliquez la différence avec .then()/.catch().
  5. Bonus : Implémentez un cache simple : si les données ont déjà été chargées (stockées en variable), ne refaites pas l'appel réseau.

📝 Exercice 4 — POO et Modules ES6 Avancé

  1. Créez une classe BankAccount avec propriétés privées (#balance), méthodes deposit(), withdraw(), getHistory() et un getter balance.
  2. Étendez avec SavingsAccount extends BankAccount qui ajoute un taux d'intérêt et une méthode applyInterest().
  3. Créez un module bank.js qui exporte les classes, et un app.js qui les importe et les utilise (type="module").
  4. Ajoutez une méthode statique BankAccount.transfer(from, to, amount) qui gère le transfert entre deux comptes avec validation.

Travaux Pratiques

🔬 TP 1 — Application météo avec API OpenWeatherMap Moyen — 2h

Objectif : Construire une application météo qui interroge une vraie API REST.

  1. Créez un compte gratuit sur openweathermap.org et obtenez une clé API.
  2. Interface HTML : champ de saisie ville + bouton rechercher + zone d'affichage des résultats.
  3. Appel API : https://api.openweathermap.org/data/2.5/weather?q={ville}&appid={KEY}&units=metric&lang=fr.
  4. Affichez : nom de la ville, pays, température actuelle, ressenti, description, humidité, vent, icône météo.
  5. Géolocalisation : bouton "Ma position" qui utilise navigator.geolocation pour détecter la ville automatiquement.
  6. Historique : stockez les 5 dernières recherches dans localStorage et affichez-les comme suggestions.

🔬 TP 2 — Jeu du Snake en Canvas Avancé — 3h

Objectif : Implémenter le jeu classique Snake avec HTML Canvas et JavaScript pur.

  1. Canvas : Créer le canvas 400×400, définir la taille des cellules (20px), dessiner la grille.
  2. Serpent : Représenter comme un tableau de segments [{x,y}]. Déplacer en ajoutant la tête et supprimant la queue.
  3. Contrôles : Clavier (flèches) + swipe tactile sur mobile. Interdire le demi-tour immédiat.
  4. Pomme : Générer en position aléatoire (hors serpent). Incrémenter le score. Accélérer progressivement.
  5. Collisions : Mur → Game Over. Collision avec soi-même → Game Over. Afficher un écran "Game Over" avec le score et un bouton Rejouer.
  6. Bonus : High score en localStorage, effets visuels (flash lors de la mort), sons avec Web Audio API.
