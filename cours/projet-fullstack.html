PROJ.03
2. Backend FlaskAPI REST complète
Python — backend/app.py 
import pymysql
from flask import Flask, jsonify, request
from flask_cors import CORS

app = Flask(__name__)
CORS(app)

DB = {
    'host': 'localhost', 
    'user': 'root',
    'password': 'votre_mdp', 
    'database': 'gestion_eleves',
    'charset': 'utf8mb4', 
    'cursorclass': pymysql.cursors.DictCursor
}

def db(): 
    return pymysql.connect(**DB)

# GET /api/eleves — lire tous les élèves
@app.route('/api/eleves')
def liste():
    c = db()
    with c.cursor() as cur: 
        cur.execute("SELECT * FROM eleves")
        r = cur.fetchall()
    c.close()
    return jsonify(r)

# POST /api/eleves — créer un élève
@app.route('/api/eleves', methods=['POST'])
def creer():
    d = request.get_json()
    c = db()
    with c.cursor() as cur:
        cur.execute(
            "INSERT INTO eleves (nom,prenom,classe,note) VALUES (%s,%s,%s,%s)",
            (d['nom'], d['prenom'], d.get('classe',''), d.get('note',0))
        )
        c.commit()
        nid = cur.lastrowid
    c.close()
    return jsonify({"id": nid}), 201

# DELETE /api/eleves/<id>
@app.route('/api/eleves/<int:id>', methods=['DELETE'])
def supprimer(id):
    c = db()
    with c.cursor() as cur:
        cur.execute("DELETE FROM eleves WHERE id=%s", (id,))
        c.commit()
    c.close()
    return jsonify({"ok": True})

if __name__ == '__main__':
    app.run(debug=True, host='0.0.0.0', port=5000)

JavaScript — frontend/script.js 
// ⚠️ Remplacer 192.168.1.10 par l'IP du PC serveur
const API = 'http://192.168.1.10:5000/api';

// Charger et afficher les élèves
async function charger() {
  const eleves = await (await fetch(`${API}/eleves`)).json();
  const tbody  = document.getElementById('corps');
  tbody.innerHTML = eleves.map(e => `
    <tr>
      <td>${e.id}</td><td>${e.nom}</td><td>${e.prenom}</td>
      <td>${e.classe}</td><td>${e.note}</td>
      <td><button onclick="supprimer(${e.id})">Supprimer</button></td>
    </tr>`).join('');
}

// Ajouter un élève
document.getElementById('formAjout')
  .addEventListener('submit', async e => {
    e.preventDefault();
    await fetch(`${API}/eleves`, {
      method:  'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        nom:    document.getElementById('nom').value,
        prenom: document.getElementById('prenom').value,
        classe: document.getElementById('classe').value,
        note:   +document.getElementById('note').value
      })
    });
    e.target.reset();
    charger();
  });

// Supprimer un élève
async function supprimer(id) {
  if (!confirm('Supprimer cet élève ?')) return;
  await fetch(`${API}/eleves/${id}`, { method: 'DELETE' });
  charger();
}

// Charger au démarrage
charger();

PROJ.05
4. Déploiement réseau localWindows 11 — accessible à toute la classe
Principe : le PC du professeur (ou un PC dédié) fait office de serveur. Tous les élèves du réseau local (même Wi-Fi / même switch) accèdent à l'application via l'adresse IP du serveur.
Étapes — PC Serveur (Windows 11) 
══ ÉTAPE 1 — Trouver l'adresse IP du serveur ══════════
# Dans PowerShell ou CMD :
ipconfig
# Repérer "Adresse IPv4" → ex : 192.168.1.10

══ ÉTAPE 2 — Installer les prérequis ════════════════
# Installer Python 3.11+ depuis python.org
# Installer MySQL Community Server depuis dev.mysql.com

══ ÉTAPE 3 — Préparer la base de données ════════════
# Ouvrir MySQL Workbench ou le terminal MySQL :
mysql -u root -p
# Puis exécuter le fichier sql/init.sql
source C:/mon_projet/sql/init.sql

══ ÉTAPE 4 — Lancer le serveur Flask ════════════════
cd C:\mon_projet
.\.venv\Scripts\Activate.ps1
cd backend
python app.py
# → "Running on http://0.0.0.0:5000"

══ ÉTAPE 5 — Ouvrir le pare-feu Windows ════════════
# PowerShell (admin) :
New-NetFirewallRule -DisplayName "Flask 5000" -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow

══ ÉTAPE 6 — Servir le frontend ═════════════════════
# Dans un 2ème terminal :
cd C:\mon_projet\frontend
python -m http.server 8080
# → Accessible sur http://192.168.1.10:8080

══ ÉTAPE 7 — Connexion depuis les PC élèves ═════════
# Ouvrir un navigateur et saisir :
# http://192.168.1.10:8080   ← application
# http://192.168.1.10:5000/api/eleves ← API directe

Dépannage

Problème Solution
Connexion refusée (ECONNREFUSED) Flask n'est pas lancé, ou pare-feu bloque le port 5000
Erreur CORS Vérifier que flask-cors est installé et CORS(app) présent
Page blanche Serveur HTTP (python -m http.server) pas lancé
Erreur MySQL 1045 Mauvais mot de passe dans config.py
ModuleNotFoundError .venv non activé — lancer Activate.ps1 d'abord
IP introuvable Vérifier que serveur et clients sont sur le même réseau

Exercices & Travaux PratiquesConception et développement d'une application web complète

Le projet Full Stack est l'aboutissement de votre formation. Ces exercices préparatoires et le TP final vous permettent d'intégrer toutes les compétences acquises dans une application web professionnelle et déployée.

📝 Exercice 1 — Architecture MVC Flask Moyen

  1. Restructurez une application Flask monolithique en utilisant les Blueprints : auth, api, admin, main — chacun dans son propre dossier avec ses routes, templates et modèles.
  2. Implémentez une Application Factory (create_app(config)) permettant de créer l'application avec différentes configurations (développement, test, production).
  3. Ajoutez la gestion des erreurs globale : pages d'erreur personnalisées pour 404, 403, 500 et gestionnaire d'exceptions JSON pour l'API.

📝 Exercice 2 — Tests automatisés Avancé

  1. Écrivez des tests unitaires avec pytest pour vos modèles SQLAlchemy : création, lecture, mise à jour, suppression, et validation des contraintes.
  2. Écrivez des tests d'intégration pour vos routes Flask : testez chaque endpoint avec le client de test Flask (app.test_client()).
  3. Mesurez la couverture de tests avec pytest-cov. Visez > 80% de couverture.
  4. Ajoutez des tests end-to-end avec Selenium ou Playwright pour simuler un utilisateur réel navigant sur votre site.

🔬 TP FINAL — Application Full Stack de gestion CIEL Avancé — 8h (projet noté)

Objectif : Développer et déployer une application web de gestion pour la filière CIEL (gestion des élèves, des projets IoT et des ressources).

Fonctionnalités requises :

  1. Authentification : Inscription, connexion, rôles (admin, professeur, élève) avec Flask-Login. Réinitialisation de mot de passe par email.
  2. Gestion des élèves : CRUD complet, import CSV, export PDF des bulletins, photos de profil uploadées et redimensionnées avec Pillow.
  3. Projets IoT : Chaque projet peut avoir un statut, une description, des membres, des fichiers attachés et un historique de modifications.
  4. Dashboard : Statistiques en temps réel (Chart.js), activité récente, alertes.
  5. API REST : Endpoints JSON documentés pour l'intégration avec les projets Arduino/ESP32 (envoi de données capteurs vers la base de données).
  6. Déploiement : Déploiement sur un VPS ou Render/Railway avec HTTPS, variables d'environnement sécurisées et base de données PostgreSQL.
  7. Livrable : Code sur GitHub (README complet, instructions d'installation, variables d'env documentées) + URL de l'application déployée + rapport de 5 pages.
🏆
Critères d'évaluation : Fonctionnalité (40%) + Qualité du code / bonnes pratiques (25%) + Design et UX (15%) + Déploiement et documentation (20%).
