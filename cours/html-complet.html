H.01
Introduction à HTMLQu'est-ce que HTML ? Comment fonctionne le web ?

Qu'est-ce que HTML ?

HTML (HyperText Markup Language) est le langage de balisage qui structure le contenu des pages web. Ce n'est pas un langage de programmation — il ne contient pas de logique — mais un langage de description.

Chaque élément HTML est défini par une balise (tag) qui indique au navigateur comment afficher le contenu.

Analogie : Si une page web était un corps humain, HTML serait le squelette (structure), CSS serait la peau et les vêtements (apparence), et JavaScript serait les muscles (mouvement).

Anatomie d'une balise HTML

Anatomie d'une balise 
<!-- Structure d'une balise avec contenu -->
<balise attribut="valeur">Contenu de la balise</balise>

<!-- Exemple concret -->
<a href="https://google.com" target="_blank">Aller sur Google</a>

<!-- Balise auto-fermante (sans contenu) -->
<img src="photo.jpg" alt="Description">
<br>
<hr>
<input type="text">
Élément Description Exemple
Balise ouvrante Début de l'élément <p>
Balise fermante Fin de l'élément (avec /) </p>
Attribut Information supplémentaire href="..."
Valeur Valeur de l'attribut (entre guillemets) "https://..."
Contenu Texte ou autres balises Mon texte

Structure minimale d'une page HTML5

HTML — Structure de base 
<!DOCTYPE html>                     <!-- Déclaration HTML5 -->
<html lang="fr">                     <!-- Élément racine + langue -->
<head>                              <!-- Métadonnées (invisible) -->
  <meta charset="UTF-8">             <!-- Encodage des caractères -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Description de la page">
  <title>Titre de la page</title>    <!-- Titre onglet navigateur -->
  <link rel="stylesheet" href="style.css">
</head>
<body>                              <!-- Contenu visible -->
  
  <h1>Mon premier titre</h1>
  <p>Mon premier paragraphe.</p>
  
  <script src="script.js"></script>  <!-- JS en fin de body -->
</body>
</html>
Important : Toujours déclarer <!DOCTYPE html> en première ligne et lang="fr" pour l'accessibilité et le référencement.
H.02
Balises de texteTitres, paragraphes, mise en forme du texte

Les titres (h1 à h6)

HTML propose 6 niveaux de titres, du plus important (<h1>) au moins important (<h6>). Ils définissent la hiérarchie du contenu.

HTML — Titres 
<h1>Titre principal (un seul par page)</h1>
<h2>Titre de section</h2>
<h3>Sous-section</h3>
<h4>Sous-sous-section</h4>
<h5>Niveau 5</h5>
<h6>Niveau 6 (le plus petit)</h6>
SEO : Utilisez un seul <h1> par page (le titre principal). Les moteurs de recherche l'utilisent pour comprendre le sujet de la page.

Paragraphes et sauts de ligne

HTML — Paragraphes 
<!-- Paragraphe : bloc de texte -->
<p>Ceci est un paragraphe. Le navigateur ajoute automatiquement
   un espace avant et après.</p>

<p>Un autre paragraphe séparé du premier.</p>

<!-- Saut de ligne (sans nouveau paragraphe) -->
<p>Première ligne<br>
   Deuxième ligne (même paragraphe)</p>

<!-- Ligne horizontale de séparation -->
<hr>

<!-- Texte préformaté (conserve espaces et retours) -->
<pre>
  Ce texte garde
    son formatage
      exact.
</pre>

Mise en forme du texte

Balise Effet Exemple de code Rendu
<strong> Texte important (gras) <strong>Important</strong> Important
<em> Emphase (italique) <em>Emphase</em> Emphase
<b> Gras (sans importance) <b>Gras</b> Gras
<i> Italique (terme technique) <i>variable</i> variable
<u> Souligné <u>Souligné</u> Souligné
<s> Barré <s>Barré</s> Barré
<mark> Surligné <mark>Surligné</mark> Surligné
<small> Petit texte <small>Petit</small> Petit
<sup> Exposant x<sup>2</sup> x2
<sub> Indice H<sub>2</sub>O H2O
<code> Code informatique <code>let x = 5;</code> let x = 5;
<kbd> Touche clavier <kbd>Ctrl</kbd> Ctrl
<abbr> Abréviation <abbr title="HyperText">HTML</abbr> HTML
<q> Citation courte <q>Citation</q> Citation
<blockquote> Citation longue (bloc) <blockquote>...</blockquote> Bloc indenté
HTML — Exemple complet 
<p>
  En physique, la formule <strong>E = mc<sup>2</sup></strong> 
  d'<em>Albert Einstein</em> montre que l'énergie 
  est égale à la masse multipliée par la vitesse de la lumière au carré.
</p>

<p>
  Pour copier du texte, utilisez <kbd>Ctrl</kbd> + <kbd>C</kbd>.
</p>

<blockquote>
  <p>L'imagination est plus importante que le savoir.</p>
  <footer>— Albert Einstein</footer>
</blockquote>
À retenir
  • <strong> = importance sémantique (gras) • <b> = gras visuel uniquement
  • <em> = emphase sémantique (italique) • <i> = italique visuel
  • Préférez les balises sémantiques (strong, em) pour l'accessibilité
  • <br> = saut de ligne • <hr> = séparation thématique
H.03
Liens et ImagesNavigation et médias visuels

Les liens hypertextes (<a>)

La balise <a> (anchor) crée des liens cliquables vers d'autres pages, fichiers ou sections.

HTML — Liens 
<!-- Lien externe (vers un autre site) -->
<a href="https://www.google.com">Aller sur Google</a>

<!-- Ouvrir dans un nouvel onglet -->
<a href="https://www.google.com" target="_blank" rel="noopener">
  Google (nouvel onglet)
</a>

<!-- Lien interne (même site) -->
<a href="contact.html">Page contact</a>
<a href="pages/about.html">À propos</a>
<a href="../index.html">Retour accueil</a>

<!-- Lien vers une section de la page (ancre) -->
<a href="#section-contact">Aller à Contact</a>
...
<section id="section-contact">...</section>

<!-- Lien email -->
<a href="mailto:[email protected]">Envoyer un email</a>

<!-- Lien téléphone -->
<a href="tel:+33612345678">Appeler</a>

<!-- Lien de téléchargement -->
<a href="document.pdf" download>Télécharger le PDF</a>
<a href="image.png" download="mon-image.png">Télécharger l'image</a>
Attribut Valeur Description
href URL Destination du lien (obligatoire)
target _blank Ouvre dans un nouvel onglet
target _self Même fenêtre (par défaut)
rel noopener Sécurité pour _blank
download nom fichier Force le téléchargement
title texte Info-bulle au survol

Les images (<img>)

HTML — Images 
<!-- Image basique -->
<img src="photo.jpg" alt="Description de la photo">

<!-- Image avec dimensions -->
<img src="logo.png" alt="Logo entreprise" width="200" height="100">

<!-- Image depuis une URL -->
<img src="https://via.placeholder.com/300x200" alt="Placeholder">

<!-- Image dans un dossier -->
<img src="images/produit.jpg" alt="Photo produit">
<img src="../assets/icon.svg" alt="Icône">

<!-- Image cliquable (lien) -->
<a href="page.html">
  <img src="banniere.jpg" alt="Cliquez ici">
</a>

<!-- Figure avec légende -->
<figure>
  <img src="graphique.png" alt="Graphique des ventes">
  <figcaption>Figure 1 : Évolution des ventes 2024</figcaption>
</figure>
Accessibilité : L'attribut alt est obligatoire. Il décrit l'image pour les personnes malvoyantes et s'affiche si l'image ne charge pas.
Attribut Description Exemple
src Chemin ou URL de l'image (obligatoire) "photo.jpg"
alt Texte alternatif (obligatoire) "Portrait de Marie"
width Largeur en pixels "400"
height Hauteur en pixels "300"
loading Chargement différé "lazy"
title Info-bulle au survol "Cliquez pour agrandir"

Formats d'images

Format Extension Usage recommandé
JPEG .jpg, .jpeg Photos, images complexes (compression avec perte)
PNG .png Logos, images avec transparence
GIF .gif Animations simples, icônes
SVG .svg Icônes, logos vectoriels (redimensionnables)
WebP .webp Format moderne, compression optimale
H.04
Les ListesListes ordonnées, non ordonnées et de définition

Liste non ordonnée (<ul>)

Utilisée pour des éléments sans ordre particulier. Par défaut, affiche des puces.

HTML — Liste non ordonnée 
<ul>
  <li>Premier élément</li>
  <li>Deuxième élément</li>
  <li>Troisième élément</li>
</ul>

<!-- Liste imbriquée -->
<ul>
  <li>Fruits
    <ul>
      <li>Pomme</li>
      <li>Banane</li>
    </ul>
  </li>
  <li>Légumes
    <ul>
      <li>Carotte</li>
      <li>Tomate</li>
    </ul>
  </li>
</ul>

Liste ordonnée (<ol>)

Utilisée quand l'ordre des éléments est important. Affiche des numéros.

HTML — Liste ordonnée 
<!-- Liste numérotée classique -->
<ol>
  <li>Ouvrir le fichier</li>
  <li>Modifier le contenu</li>
  <li>Enregistrer</li>
</ol>

<!-- Commencer à un autre numéro -->
<ol start="5">
  <li>Cinquième étape</li>
  <li>Sixième étape</li>
</ol>

<!-- Lettres au lieu de chiffres -->
<ol type="a">
  <li>Option a</li>
  <li>Option b</li>
</ol>

<!-- Chiffres romains -->
<ol type="I">
  <li>Chapitre I</li>
  <li>Chapitre II</li>
</ol>

<!-- Ordre inversé -->
<ol reversed>
  <li>Troisième (affiché 3)</li>
  <li>Deuxième (affiché 2)</li>
  <li>Premier (affiché 1)</li>
</ol>
Attribut Valeurs Effet
type 1 Chiffres (défaut)
type a Lettres minuscules
type A Lettres majuscules
type i Romains minuscules
type I Romains majuscules
start nombre Numéro de départ
reversed (sans valeur) Ordre décroissant

Liste de définition (<dl>)

Utilisée pour des paires terme/définition (glossaires, FAQ, métadonnées).

HTML — Liste de définition 
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — structure des pages web</dd>
  
  <dt>CSS</dt>
  <dd>Cascading Style Sheets — mise en forme</dd>
  
  <dt>JavaScript</dt>
  <dd>Langage de programmation pour l'interactivité</dd>
  <dd>Peut être utilisé côté client et serveur</dd>
</dl>
H.05
Les TableauxPrésentation de données tabulaires

Structure d'un tableau

HTML — Tableau complet 
<table>
  <!-- Légende du tableau -->
  <caption>Notes des élèves - Semestre 1</caption>
  
  <!-- En-tête du tableau -->
  <thead>
    <tr>
      <th>Nom</th>
      <th>Prénom</th>
      <th>Maths</th>
      <th>Info</th>
      <th>Moyenne</th>
    </tr>
  </thead>
  
  <!-- Corps du tableau -->
  <tbody>
    <tr>
      <td>Dupont</td>
      <td>Alice</td>
      <td>15</td>
      <td>18</td>
      <td>16.5</td>
    </tr>
    <tr>
      <td>Martin</td>
      <td>Bob</td>
      <td>12</td>
      <td>14</td>
      <td>13</td>
    </tr>
  </tbody>
  
  <!-- Pied du tableau -->
  <tfoot>
    <tr>
      <td colspan="4">Moyenne classe</td>
      <td>14.75</td>
    </tr>
  </tfoot>
</table>
Balise Rôle
<table> Conteneur du tableau
<caption> Titre/légende du tableau
<thead> En-tête (titres des colonnes)
<tbody> Corps (données)
<tfoot> Pied (totaux, résumés)
<tr> Ligne (table row)
<th> Cellule d'en-tête (gras, centré)
<td> Cellule de données

Fusion de cellules

HTML — Fusion de cellules 
<table>
  <tr>
    <!-- colspan : fusion horizontale (colonnes) -->
    <th colspan="3">Emploi du temps</th>
  </tr>
  <tr>
    <th>Lundi</th>
    <th>Mardi</th>
    <th>Mercredi</th>
  </tr>
  <tr>
    <!-- rowspan : fusion verticale (lignes) -->
    <td rowspan="2">Cours de maths</td>
    <td>TP Info</td>
    <td>Anglais</td>
  </tr>
  <tr>
    <td>Physique</td>
    <td>Libre</td>
  </tr>
</table>
Rappel : colspan="2" = la cellule s'étend sur 2 colonnes. rowspan="3" = la cellule s'étend sur 3 lignes.

Les FormulairesSaisie de données utilisateur

Structure d'un formulaire

HTML — Formulaire complet 
<form action="/traitement.php" method="POST">
  
  <!-- Champ texte -->
  <label for="nom">Nom :</label>
  <input type="text" id="nom" name="nom" 
         placeholder="Votre nom" required>
  
  <!-- Email -->
  <label for="email">Email :</label>
  <input type="email" id="email" name="email" 
         placeholder="[email protected]" required>
  
  <!-- Mot de passe -->
  <label for="mdp">Mot de passe :</label>
  <input type="password" id="mdp" name="mdp" 
         minlength="8" required>
  
  <!-- Nombre -->
  <label for="age">Âge :</label>
  <input type="number" id="age" name="age" 
         min="0" max="120" value="18">
  
  <!-- Date -->
  <label for="date">Date de naissance :</label>
  <input type="date" id="date" name="date_naissance">
  
  <!-- Bouton de soumission -->
  <button type="submit">Envoyer</button>
  
</form>

  <option value="Firefox">
  <option value="Edge">

Attribut Description Exemple

À retenir — Formulaires

  • required rend un champ obligatoire (validation côté client)
H.07
Balises Sémantiques HTML5Structure significative pour l'accessibilité et le SEO

Pourquoi la sémantique ?

Les balises sémantiques décrivent le rôle du contenu, pas seulement son apparence. Elles améliorent l'accessibilité (lecteurs d'écran), le SEO (référencement) et la maintenabilité du code.

Ancienne méthode (non sémantique)
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="footer">...</div>
HTML5 sémantique
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

Les balises sémantiques HTML5

HTML — Structure sémantique complète 
<body>
  <!-- En-tête du site (logo, titre, navigation principale) -->
  <header>
    <img src="logo.png" alt="Logo">
    <h1>Mon Site</h1>
    
    <!-- Navigation principale -->
    <nav>
      <ul>
        <li><a href="/">Accueil</a></li>
        <li><a href="/about">À propos</a></li>
        <li><a href="/contact">Contact</a></li>
      </ul>
    </nav>
  </header>
  
  <!-- Contenu principal (unique par page) -->
  <main>
    
    <!-- Article autonome (blog post, news) -->
    <article>
      <header>
        <h2>Titre de l'article</h2>
        <time datetime="2024-01-15">15 janvier 2024</time>
      </header>
      
      <!-- Section thématique -->
      <section>
        <h3>Introduction</h3>
        <p>Contenu de l'introduction...</p>
      </section>
      
      <section>
        <h3>Développement</h3>
        <p>Contenu principal...</p>
        
        <!-- Figure avec légende -->
        <figure>
          <img src="graphique.png" alt="Graphique">
          <figcaption>Figure 1 : Statistiques 2024</figcaption>
        </figure>
      </section>
      
      <footer>
        <p>Auteur : Alice Dupont</p>
      </footer>
    </article>
    
  </main>
  
  <!-- Contenu secondaire (sidebar) -->
  <aside>
    <h3>Articles récents</h3>
    <ul>
      <li><a href="#">Article 1</a></li>
      <li><a href="#">Article 2</a></li>
    </ul>
  </aside>
  
  <!-- Pied de page -->
  <footer>
    <p>© 2024 Mon Site - Tous droits réservés</p>
    <address>
      Contact : <a href="mailto:[email protected]">[email protected]</a>
    </address>
  </footer>
  
</body>
Balise Rôle Usage
<header> En-tête Logo, titre, navigation (page ou section)
<nav> Navigation Menu principal, fil d'Ariane
<main> Contenu principal Un seul par page
<article> Contenu autonome Blog post, commentaire, widget
<section> Section thématique Groupe de contenu avec titre
<aside> Contenu secondaire Sidebar, encarts, publicités
<footer> Pied de page Copyright, liens, contact
<figure> Illustration Image, graphique avec légende
<figcaption> Légende Description de la figure
<time> Date/heure Publication, événement
<address> Contact Email, adresse postale
<mark> Surligné Texte important
<details> Accordéon Contenu repliable
<summary> Titre accordéon Titre cliquable de details

Exemple : Accordéon avec details/summary

HTML — Accordéon natif 
<details>
  <summary>Cliquez pour voir plus</summary>
  <p>Ce contenu est caché par défaut et s'affiche au clic.</p>
  <p>Aucun JavaScript nécessaire !</p>
</details>

<!-- Ouvert par défaut -->
<details open>
  <summary>FAQ - Question fréquente</summary>
  <p>Réponse à la question...</p>
</details>
À retenir — Sémantique HTML5
  • Utilisez <header>, <main>, <footer> au lieu de <div>
  • <main> doit être unique par page
  • <article> = contenu autonome qui a du sens seul
  • <section> = groupe de contenu thématique (avec un titre)
  • La sémantique améliore l'accessibilité et le référencement

Exercices & Travaux PratiquesMettre en pratique le HTML de la structure de base aux formulaires avancés

Le HTML est le squelette de toute page web. Ces exercices vous permettent de passer de la théorie à la pratique, en construisant progressivement des pages de plus en plus complètes et sémantiques.

💡
Outil recommandé : Utilisez VS Code avec l'extension Live Server pour voir vos modifications en temps réel dans le navigateur.

Exercices de compréhension

📝 Exercice 1 — Structure minimale valide Facile

  1. Créez un fichier index.html avec la structure HTML5 complète (DOCTYPE, html, head, body).
  2. Dans le head : ajoutez le charset UTF-8, le viewport pour mobile, un titre de page et une balise meta description.
  3. Dans le body : créez un titre h1, un sous-titre h2, un paragraphe de présentation et une liste à puces de 5 éléments décrivant vos compétences.
  4. Validation : Collez votre code dans le validateur W3C et corrigez toutes les erreurs.

📝 Exercice 2 — Tableaux de données Moyen

  1. Créez un tableau HTML représentant un emploi du temps scolaire (5 jours × 6 créneaux).
  2. Utilisez thead, tbody, tfoot, th scope="col" et th scope="row" correctement.
  3. Fusionnez des cellules avec colspan et rowspan pour les cours qui durent 2 heures.
  4. Ajoutez une légende (caption) et des données de résumé dans le tfoot (nombre total d'heures par matière).
  5. Accessibilité : Ajoutez l'attribut summary et vérifiez que le tableau est lisible avec un lecteur d'écran (logique de lecture ligne par ligne).

📝 Exercice 3 — Formulaire d'inscription complet Moyen

  1. Créez un formulaire d'inscription avec les champs : prénom, nom, email, mot de passe (avec confirmation), date de naissance, genre (radio), centres d'intérêt (checkbox), niveau d'études (select), photo de profil (file) et un textarea pour la biographie.
  2. Utilisez la validation HTML5 native : required, minlength, maxlength, pattern, type="email", type="date".
  3. Associez chaque champ à son label avec l'attribut for/id.
  4. Regroupez les champs avec fieldset et legend.
