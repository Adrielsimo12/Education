NET.01
Introduction aux RéseauxConcepts fondamentaux et architecture

Qu'est-ce qu'un réseau informatique ?

Un réseau informatique est un ensemble d'ordinateurs et d'équipements interconnectés qui peuvent communiquer et partager des ressources (fichiers, imprimantes, connexion Internet).

Objectifs d'un réseau : Partage de ressources, communication (email, messagerie), accès à des données centralisées, travail collaboratif.

Types de réseaux selon l'étendue

Type Nom complet Portée Exemples
PAN Personal Area Network ~10 mètres Bluetooth, smartphone ↔ écouteurs
LAN Local Area Network ~100 mètres Réseau d'entreprise, école, domicile
MAN Metropolitan Area Network ~10 km Réseau d'une ville, campus universitaire
WAN Wide Area Network Mondial Internet, réseau d'entreprise multi-sites

Équipements réseau essentiels

Switch (Commutateur)
Connecte plusieurs appareils dans un LAN
Couche 2 (liaison de données)
Routeur
Interconnecte plusieurs réseaux (ex: LAN ↔ Internet)
Couche 3 (réseau)
Point d'accès WiFi
Permet la connexion sans fil
802.11ac/ax (WiFi 5/6)
Pare-feu
Filtre le trafic entrant/sortant
Sécurité périmétrique
Points clés
  • Un réseau permet le partage de ressources et la communication
  • LAN = réseau local (entreprise, école)
  • WAN = réseau étendu (Internet)
  • Switch = connexion locale, Routeur = connexion entre réseaux
NET.02
Modèle OSILes 7 couches du modèle de référence

Qu'est-ce que le modèle OSI ?

Le modèle OSI (Open Systems Interconnection) est un modèle de référence qui décrit comment les données circulent dans un réseau en 7 couches. Chaque couche a un rôle spécifique et communique avec les couches adjacentes.

Astuce mnémotechnique (bas → haut) :
Please Do Not Throw Sausage Pizza Away
= Physique, Données, Nétwork, Transport, Session, Présentation, Application

Les 7 couches du modèle OSI

Couche Nom Rôle Exemples Unité
7 Application Interface utilisateur, services HTTP, FTP, SMTP, DNS Données
6 Présentation Formatage, chiffrement, compression SSL/TLS, JPEG, ASCII Données
5 Session Gestion des sessions (ouverture/fermeture) NetBIOS, RPC Données
4 Transport Transport fiable des données TCP, UDP Segment
3 Réseau Adressage logique et routage IP, ICMP, Routeurs Paquet
2 Liaison de données Adressage physique (MAC), détection d'erreurs Ethernet, WiFi, Switch Trame
1 Physique Transmission des bits sur le support Câbles, fibres, ondes radio Bit

Exemple de transmission HTTP

Quand vous ouvrez une page web (http://example.com), voici ce qui se passe à chaque couche :

Encapsulation OSI 
┌─ COUCHE 7 (Application) ──────────────────────┐
│ Requête HTTP : GET /index.html HTTP/1.1       │
│ Host: example.com                              │
└────────────────────────────────────────────────┘
            ↓ Encapsulation
┌─ COUCHE 4 (Transport) ────────────────────────┐
│ [En-tête TCP: Port 80] + Données HTTP         │
└────────────────────────────────────────────────┘
            ↓
┌─ COUCHE 3 (Réseau) ───────────────────────────┐
│ [En-tête IP: @IP source/dest] + Segment TCP   │
└────────────────────────────────────────────────┘
            ↓
┌─ COUCHE 2 (Liaison) ──────────────────────────┐
│ [En-tête Ethernet: @MAC] + Paquet IP          │
└────────────────────────────────────────────────┘
            ↓
┌─ COUCHE 1 (Physique) ─────────────────────────┐
│ 010110110101... (signaux électriques/optiques)│
└────────────────────────────────────────────────┘
NET.03
Modèle TCP/IPArchitecture d'Internet

TCP/IP vs OSI

Le modèle TCP/IP (Transmission Control Protocol / Internet Protocol) est le modèle réellement utilisé sur Internet. Il est composé de 4 couches (simplification du modèle OSI).

Modèle TCP/IP (4 couches) Équivalent OSI Protocoles
4. Application Couches 5, 6, 7 HTTP, FTP, SMTP, DNS, SSH
3. Transport Couche 4 TCP, UDP
2. Internet Couche 3 IP, ICMP, ARP
1. Accès réseau Couches 1, 2 Ethernet, WiFi

TCP vs UDP

Critère TCP (fiable) UDP (rapide)
Connexion Orienté connexion (3-way handshake) Sans connexion
Fiabilité Garantit la livraison (accus Aucune garantie
Ordre Ordre garanti Ordre non garanti
Vitesse Plus lent Très rapide
Usage HTTP, FTP, Email Streaming vidéo, jeux en ligne, DNS

Établissement de connexion TCP (3-way handshake)

Schéma 
Client                          Serveur
  |                                 |
  |  ──── SYN (seq=100) ────────>   |  1. Demande de connexion
  |                                 |
  |  <─── SYN-ACK (ack=101) ─────   |  2. Accord + acquittement
  |                                 |
  |  ──── ACK (ack=1) ───────────>  |  3. Confirmation finale
  |                                 |
  | Connexion établie ✓             |
  |  ──── Données HTTP... ──────>   |
NET.04
Adressage IPIPv4, IPv6, classes et masques de sous-réseau

Qu'est-ce qu'une adresse IP ?

Une adresse IP (Internet Protocol) est un identifiant numérique unique attribué à chaque appareil connecté à un réseau. Elle permet de localiser et router les paquets de données.

IPv4
Format : 192.168.1.1 (4 octets)
~4,3 milliards d'adresses
IPv6
Format : 2001:0db8::1 (128 bits)
340 sextillions d'adresses

Classes d'adresses IPv4

Classe Plage Masque par défaut Usage
A 1.0.0.0 → 126.255.255.255 255.0.0.0 (/8) Très grands réseaux
B 128.0.0.0 → 191.255.255.255 255.255.0.0 (/16) Réseaux moyens
C 192.0.0.0 → 223.255.255.255 255.255.255.0 (/24) Petits réseaux
D 224.0.0.0 → 239.255.255.255 — Multicast
E 240.0.0.0 → 255.255.255.255 — Réservé (expérimental)

Adresses IP privées (RFC 1918)

Plages réservées pour les réseaux locaux (non routables sur Internet) :

  • Classe A : 10.0.0.0 → 10.255.255.255 (/8)
  • Classe B : 172.16.0.0 → 172.31.255.255 (/12)
  • Classe C : 192.168.0.0 → 192.168.255.255 (/16)

Masque de sous-réseau (Subnet Mask)

Le masque détermine quelle partie de l'adresse IP représente le réseau et quelle partie représente l'hôte.

Exemple 
Adresse IP :    192.168.1.10
Masque :        255.255.255.0  (ou /24 en notation CIDR)

┌────────────┬─────┐
│  Réseau    │ Hôte│
│ 192.168.1  │  10 │
└────────────┴─────┘

Adresse réseau :     192.168.1.0
Adresse broadcast :  192.168.1.255
Plage utilisable :   192.168.1.1 → 192.168.1.254
Nombre d'hôtes :     254 (2⁸ - 2)

Commandes réseau utiles

Bash / CMD 
# Afficher la configuration IP (Linux)
ip addr show
ifconfig

# Afficher la configuration IP (Windows)
ipconfig /all

# Tester la connectivité
ping 8.8.8.8

# Tracer le chemin réseau
traceroute google.com  # Linux
tracert google.com     # Windows

# Afficher la table de routage
ip route show          # Linux
route print            # Windows
NET.05
DNS (Domain Name System)Résolution de noms de domaine

Qu'est-ce que le DNS ?

Le DNS est un système qui traduit les noms de domaine (google.com) en adresses IP (142.250.185.78). C'est "l'annuaire téléphonique d'Internet".

Analogie : DNS = Annuaire téléphonique. Vous cherchez "Restaurant La Fontaine" et l'annuaire vous donne le numéro de téléphone.

Processus de résolution DNS

Étapes 
1. Vous tapez "www.google.com" dans le navigateur
   ↓
2. Le navigateur vérifie son cache DNS local
   ↓
3. Si non trouvé → interroge le résolveur DNS (FAI)
   ↓
4. Résolveur vérifie son cache
   ↓
5. Si non trouvé → interroge le serveur racine (.)
   ↓
6. Serveur racine renvoie l'adresse du serveur TLD (.com)
   ↓
7. Serveur TLD renvoie l'adresse du serveur autoritatif (google.com)
   ↓
8. Serveur autoritatif renvoie l'adresse IP : 142.250.185.78
   ↓
9. Navigateur se connecte à cette IP

Types d'enregistrements DNS

Type Description Exemple
A Adresse IPv4 example.com → 93.184.216.34
AAAA Adresse IPv6 example.com → 2606:2800:220:1:...
CNAME Alias (nom canonique) www.example.com → example.com
MX Serveur de messagerie example.com → mail.example.com
TXT Texte (vérification domaine) SPF, DKIM
NS Serveur de noms autoritatif ns1.example.com

Commandes DNS

Bash 
# Résoudre un nom de domaine
nslookup google.com
dig google.com

# Voir tous les enregistrements DNS
dig google.com ANY

# Interroger un serveur DNS spécifique
dig @8.8.8.8 google.com

# Vider le cache DNS (Windows)
ipconfig /flushdns

# Vider le cache DNS (Linux)
sudo systemd-resolve --flush-caches
NET.06
DHCPAttribution automatique d'adresses IP

Qu'est-ce que DHCP ?

Le DHCP (Dynamic Host Configuration Protocol) est un protocole qui attribue automatiquement des adresses IP aux appareils qui se connectent au réseau. Sans DHCP, il faudrait configurer manuellement chaque appareil.

Processus DHCP (DORA)

Échanges DHCP 
Client                          Serveur DHCP
  |                                    |
  | ─ DISCOVER (broadcast) ───────>  |  1. Recherche serveur DHCP
  |                                    |
  | <─ OFFER (IP proposée) ─────────  |  2. Offre d'une IP
  |                                    |
  | ─ REQUEST (acceptation) ───────>  |  3. Demande officielle
  |                                    |
  | <─ ACK (confirmation) ───────────  |  4. Confirmation (bail accordé)
  |                                    |
  | Configuration terminée ✓           |

Informations fournies par DHCP

  • Adresse IP du client (ex: 192.168.1.50)
  • Masque de sous-réseau (ex: 255.255.255.0)
  • Passerelle par défaut (routeur, ex: 192.168.1.1)
  • Serveur DNS (ex: 8.8.8.8)
  • Durée du bail (ex: 24 heures)
NET.07
RoutageComment les données trouvent leur chemin

Qu'est-ce que le routage ?

Le routage est le processus par lequel les routeurs déterminent le meilleur chemin pour acheminer les paquets de données d'une source à une destination à travers plusieurs réseaux.

Types de routage

Type Description Usage
Routage statique Routes configurées manuellement par l'administrateur Petits réseaux simples
Routage dynamique Routes apprises automatiquement via des protocoles Grands réseaux complexes

Protocoles de routage dynamique

RIP
Routing Information Protocol
Vecteur de distance, max 15 sauts
OSPF
Open Shortest Path First
État de lien, calcul du plus court chemin
BGP
Border Gateway Protocol
Routage entre systèmes autonomes (Internet)

Configuration routage statique (Linux)

Bash 
# Ajouter une route statique
sudo ip route add 10.0.0.0/24 via 192.168.1.1 dev eth0

# Supprimer une route
sudo ip route del 10.0.0.0/24

# Afficher la table de routage
ip route show

# Route par défaut (gateway)
sudo ip route add default via 192.168.1.1
NET.08
VLANRéseaux locaux virtuels

Qu'est-ce qu'un VLAN ?

Un VLAN (Virtual Local Area Network) permet de segmenter logiquement un réseau physique en plusieurs réseaux virtuels indépendants. Chaque VLAN se comporte comme un réseau distinct.

Avantages : Sécurité (isolation du trafic), Performance (réduction du broadcast), Organisation (regroupement logique par département).

Exemple d'architecture VLAN

Schéma 
┌────────────────────────────────────────┐
│          Switch (1 seul physique)       │
├────────────────────────────────────────┤
│ VLAN 10 : Administration (192.168.10.0/24) │
│   ├─ PC Admin 1                         │
│   └─ PC Admin 2                         │
├────────────────────────────────────────┤
│ VLAN 20 : Commercial (192.168.20.0/24)    │
│   ├─ PC Commercial 1                    │
│   └─ PC Commercial 2                    │
├────────────────────────────────────────┤
│ VLAN 30 : Invités (192.168.30.0/24)       │
│   ├─ PC Invité 1                        │
│   └─ PC Invité 2                        │
└────────────────────────────────────────┘

⚠️ Les VLANs ne peuvent PAS communiquer entre eux sans routeur

TP 1 : Calcul de Sous-Réseaux (Subnetting)

Objectif : Diviser un réseau en sous-réseaux

Énoncé : Vous disposez du réseau 192.168.10.0/24. Divisez-le en 4 sous-réseaux de taille égale. Déterminez pour chaque sous-réseau :
  • Adresse réseau
  • Première IP utilisable
  • Dernière IP utilisable
  • Adresse de broadcast
  • Nouveau masque
Solution :

4 sous-réseaux = 2² → besoin de 2 bits supplémentaires
Nouveau masque : /24 + 2 = /26 (255.255.255.192)
Chaque sous-réseau : 2⁶ = 64 adresses (62 utilisables)

Sous-réseau 1 :
• Réseau : 192.168.10.0/26
• Première IP : 192.168.10.1
• Dernière IP : 192.168.10.62
• Broadcast : 192.168.10.63

Sous-réseau 2 :
• Réseau : 192.168.10.64/26
• Première IP : 192.168.10.65
• Dernière IP : 192.168.10.126
• Broadcast : 192.168.10.127

Sous-réseau 3 :
• Réseau : 192.168.10.128/26
• Première IP : 192.168.10.129
• Dernière IP : 192.168.10.190
• Broadcast : 192.168.10.191

Sous-réseau 4 :
• Réseau : 192.168.10.192/26
• Première IP : 192.168.10.193
• Dernière IP : 192.168.10.254
• Broadcast : 192.168.10.255

TP 2 : Configuration d'un Serveur DHCP

Objectif : Installer et configurer un serveur DHCP sur Linux

Énoncé : Configurez un serveur DHCP qui attribue des IP dans la plage 192.168.1.100 → 192.168.1.200 avec un bail de 12 heures.
Solution Bash 
# 1. Installer le serveur DHCP
sudo apt update
sudo apt install isc-dhcp-server

# 2. Éditer la configuration
sudo nano /etc/dhcp/dhcpd.conf

# 3. Ajouter la configuration suivante :
subnet 192.168.1.0 netmask 255.255.255.0 {
  range 192.168.1.100 192.168.1.200;
  option routers 192.168.1.1;
  option subnet-mask 255.255.255.0;
  option domain-name-servers 8.8.8.8, 8.8.4.4;
  default-lease-time 43200;  # 12 heures
  max-lease-time 86400;      # 24 heures
}

# 4. Spécifier l'interface réseau
sudo nano /etc/default/isc-dhcp-server
# Ajouter : INTERFACESv4="eth0"

# 5. Démarrer le service
sudo systemctl start isc-dhcp-server
sudo systemctl enable isc-dhcp-server

# 6. Vérifier l'état
sudo systemctl status isc-dhcp-server

# 7. Voir les baux attribués
sudo cat /var/lib/dhcp/dhcpd.leases

TP 3 : Diagnostic Réseau avec Wireshark

Objectif : Analyser le trafic réseau avec Wireshark

Énoncé : Capturez et analysez une requête HTTP avec Wireshark
Marche à suivre 
1. Installer Wireshark
   sudo apt install wireshark

2. Lancer Wireshark (en root ou avec sudo)
   sudo wireshark

3. Sélectionner l'interface réseau (ex: eth0, wlan0)

4. Démarrer la capture (bouton bleu en haut à gauche)

5. Ouvrir un navigateur et aller sur http://example.com

6. Arrêter la capture

7. Filtrer le trafic HTTP :
   Entrer dans la barre de filtre : http

8. Analyser la requête HTTP :
   - Clic droit sur un paquet HTTP GET
   - Follow → TCP Stream
   
   Vous verrez :
   GET /index.html HTTP/1.1
   Host: example.com
   User-Agent: Mozilla/5.0...
   Accept: text/html...

9. Observer les couches OSI :
   - Frame (Physique)
   - Ethernet II (Liaison)
   - IPv4 (Réseau)
   - TCP (Transport)
   - HTTP (Application)
Filtres Wireshark utiles :
• ip.addr == 192.168.1.1 : trafic d'une IP spécifique
• tcp.port == 80 : trafic HTTP
• dns : requêtes DNS uniquement
• icmp : paquets ping

Exercices & Travaux PratiquesAdressage IP, sous-réseaux, configuration et analyse de trafic

Ces exercices pratiques couvrent l'ensemble des notions réseau : adressage IP, calcul de sous-réseaux, configuration d'équipements et analyse de trafic avec Wireshark.

Exercices de compréhension

📝 Exercice 1 — Adressage IP et sous-réseaux Facile

  1. Pour l'adresse 192.168.10.45/26 : calculez l'adresse réseau, l'adresse de broadcast, la plage d'hôtes utilisables, le nombre max d'hôtes.
  2. Découpez le réseau 10.0.0.0/8 en sous-réseaux pour accueillir : 4 sites de 500 hôtes chacun. Utilisez le VLSM (découpage à taille variable). Donnez le plan d'adressage complet.
  3. Convertissez : 192.168.1.100 en binaire, 11000000.10101000.00000001.01100100 en décimal pointé, 255.255.255.192 en notation CIDR.
  4. Expliquez la différence entre adresses privées (RFC 1918) et adresses publiques. Quelles plages sont privées ?

📝 Exercice 2 — Configuration réseau Linux Moyen

  1. Avec les commandes ip addr, ip route, ping, traceroute, nslookup, ss -tulnp : diagnostiquez la connectivité d'une machine et expliquez chaque résultat.
  2. Configurez une interface réseau statique avec nmcli ou en éditant /etc/network/interfaces : IP 192.168.1.50/24, passerelle 192.168.1.1, DNS 8.8.8.8 et 1.1.1.1.
  3. Configurez un serveur DHCP minimal avec isc-dhcp-server qui distribue des adresses dans la plage 192.168.1.100-200 avec bail de 24h.
  4. Mettez en place des règles iptables pour : bloquer tout le trafic entrant sauf SSH (port 22) et HTTP/HTTPS (80/443), autoriser tout le trafic sortant.

📝 Exercice 3 — Analyse Wireshark Avancé

  1. Capturez le trafic pendant une navigation web. Identifiez dans Wireshark : une requête DNS, une handshake TCP (SYN, SYN-ACK, ACK), une requête HTTP GET, une réponse HTTP 200.
  2. Utilisez les filtres Wireshark : http.request.method == "GET", tcp.flags.syn == 1, ip.addr == 8.8.8.8.
  3. Analysez un fichier PCAP fourni par le formateur et répondez : quelle IP communique le plus ? Quel protocole est le plus utilisé ? Y a-t-il des tentatives de scan de ports ?

Travaux Pratiques

🔬 TP — Maquette réseau avec machines virtuelles Avancé — 3h

Objectif : Simuler une infrastructure réseau d'entreprise avec VirtualBox ou VMware.

  1. Topologie : 3 VMs Ubuntu : un routeur/passerelle (2 interfaces réseau), un serveur (192.168.1.10/24) et un client (192.168.1.20/24).
  2. Routage : Activer le forwarding IP sur le routeur (echo 1 > /proc/sys/net/ipv4/ip_forward). Configurer les routes statiques.
  3. Services : Installer et configurer Apache sur le serveur. Accéder au site depuis le client via IP. Puis configurer le DNS pour accéder via nom (www.monsite.local).
  4. Sécurité : Configurer ufw/iptables sur le serveur pour n'autoriser que SSH, HTTP, HTTPS depuis le réseau local.
  5. Documentation : Schéma réseau, plan d'adressage, captures Wireshark des flux, procédures de configuration.

Exercice — Modèle OSI — rôle des couches 
Couche 7 — Application  : HTTP, FTP, SMTP, DNS
Couche 6 — Présentation : SSL/TLS, chiffrement, compression
Couche 5 — Session      : établissement/maintenance des sessions
Couche 4 — Transport    : TCP (fiable), UDP (rapide), ports
Couche 3 — Réseau       : IP, routage, adresses logiques
Couche 2 — Liaison      : Ethernet, WiFi, adresses MAC, switch
Couche 1 — Physique     : câbles, bits, signaux électriques

Mnémotechnique : "Après Présentation, Sa Réunion Tient"
(Application, Présentation, Session, Réseau, Transport... inversé)

Protocoles courants et leur couche :
• HTTP/HTTPS → couche 7 (Application)
• TCP/UDP    → couche 4 (Transport)
• IP/ICMP    → couche 3 (Réseau)
• Ethernet   → couche 2 (Liaison)
• WiFi 802.11 → couches 1+2
