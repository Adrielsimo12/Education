SQL.01
Introduction à MySQLQu'est-ce qu'une base de données relationnelle ?

Base de données relationnelle

Une base de données relationnelle organise les données en tables (lignes et colonnes). Les tables sont liées entre elles par des clés.

Table
Structure qui contient les données (comme un tableau Excel)
eleves, produits, commandes
Clé primaire
Identifiant unique de chaque ligne
id AUTO_INCREMENT
Clé étrangère
Référence vers une autre table
user_id → users.id
SQL.02
Création de base et tablesCREATE DATABASE, CREATE TABLE, types de données

Créer une base de données

SQL — Création 
-- Créer la base
CREATE DATABASE boutique CHARACTER SET utf8mb4;
USE boutique;

-- Table produits
CREATE TABLE produits (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    nom         VARCHAR(100) NOT NULL,
    prix        DECIMAL(8,2),
    categorie   VARCHAR(50),
    stock       INT DEFAULT 0,
    created_at  DATETIME DEFAULT NOW()
);

-- Table utilisateurs
CREATE TABLE utilisateurs (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    nom         VARCHAR(50) NOT NULL,
    email       VARCHAR(100) UNIQUE NOT NULL,
    mot_de_passe VARCHAR(255),
    actif       BOOLEAN DEFAULT TRUE
);

Types de données MySQL

Type Description Exemple
INT Entier (-2 milliards à +2 milliards) id, age, stock
BIGINT Très grand entier populations
DECIMAL(8,2) Nombre décimal précis prix: 12345.67
FLOAT Décimal approximatif coordonnées GPS
VARCHAR(n) Texte variable (max n caractères) nom, email
TEXT Texte long description, bio
DATE Date (YYYY-MM-DD) 2024-01-15
DATETIME Date et heure 2024-01-15 14:30:00
BOOLEAN Vrai/Faux (stocké comme TINYINT) actif, publié
SQL.03
Opérations CRUDINSERT, SELECT, UPDATE, DELETE

INSERT — Créer des données

SQL — INSERT 
-- Insérer une ligne
INSERT INTO produits (nom, prix, categorie, stock)
VALUES ('Laptop', 1299.99, 'Informatique', 10);

-- Insérer plusieurs lignes
INSERT INTO produits (nom, prix, categorie, stock) VALUES
  ('Souris', 29.90, 'Périphérique', 50),
  ('Clavier', 79.00, 'Périphérique', 30),
  ('Écran 27"', 349.00, 'Informatique', 15);

SELECT — Lire des données

SQL — SELECT 
-- Tout sélectionner
SELECT * FROM produits;

-- Colonnes spécifiques
SELECT nom, prix FROM produits;

-- Avec condition
SELECT * FROM produits WHERE prix < 100;
SELECT * FROM produits WHERE categorie = 'Informatique';

-- Opérateurs logiques
SELECT * FROM produits 
WHERE prix < 500 AND stock > 0;

SELECT * FROM produits 
WHERE categorie = 'Informatique' OR categorie = 'Périphérique';

-- Tri
SELECT * FROM produits ORDER BY prix ASC;   -- croissant
SELECT * FROM produits ORDER BY prix DESC;  -- décroissant

-- Limiter les résultats
SELECT * FROM produits LIMIT 10;              -- 10 premiers
SELECT * FROM produits LIMIT 10 OFFSET 20;  -- pagination

-- Recherche partielle
SELECT * FROM produits WHERE nom LIKE '%clavier%';

-- Fonctions d'agrégation
SELECT COUNT(*) FROM produits;                -- nombre de lignes
SELECT AVG(prix) FROM produits;               -- moyenne
SELECT SUM(stock) FROM produits;              -- somme
SELECT MIN(prix), MAX(prix) FROM produits;   -- min/max

UPDATE — Modifier des données

SQL — UPDATE 
-- Modifier une ligne
UPDATE produits 
SET stock = 50, prix = 1199.99
WHERE id = 1;

-- Modifier plusieurs lignes
UPDATE produits 
SET prix = prix * 0.9   -- -10% de réduction
WHERE categorie = 'Périphérique';
Toujours ajouter WHERE ! Sans clause WHERE, UPDATE modifie toutes les lignes de la table.

DELETE — Supprimer des données

SQL — DELETE 
-- Supprimer une ligne
DELETE FROM produits WHERE id = 5;

-- Supprimer avec condition
DELETE FROM produits WHERE stock = 0;

-- ⚠️ DANGER : supprimer TOUT (à éviter !)
DELETE FROM produits;  -- Vide la table entière
SQL.04
JointuresRelier plusieurs tables

INNER JOIN

SQL — Jointures 
-- Tables exemple
CREATE TABLE commandes (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT,
    produit_id  INT,
    quantite    INT,
    date_commande DATETIME DEFAULT NOW(),
    FOREIGN KEY (user_id) REFERENCES utilisateurs(id),
    FOREIGN KEY (produit_id) REFERENCES produits(id)
);

-- INNER JOIN : lignes qui existent dans les deux tables
SELECT 
    u.nom AS client,
    p.nom AS produit,
    c.quantite,
    c.date_commande
FROM commandes c
INNER JOIN utilisateurs u ON c.user_id = u.id
INNER JOIN produits p ON c.produit_id = p.id
ORDER BY c.date_commande DESC;
SQL.05
Flask + MySQLConnecter l'API à la base de données

Connexion et requêtes

Python — app.py avec MySQL 
import pymysql
from flask import Flask, jsonify, request
from flask_cors import CORS

app = Flask(__name__)
CORS(app)

DB_CONFIG = {
    'host':     'localhost',
    'user':     'root',
    'password': 'votre_mdp',
    'database': 'boutique',
    'charset':  'utf8mb4'
}

def get_db():
    return pymysql.connect(**DB_CONFIG, cursorclass=pymysql.cursors.DictCursor)

# GET /api/produits — lire tous les produits
@app.route('/api/produits')
def get_produits():
    conn = get_db()
    with conn.cursor() as cur:
        cur.execute("SELECT * FROM produits ORDER BY nom")
        produits = cur.fetchall()
    conn.close()
    return jsonify(produits)

# POST /api/produits — créer un produit
@app.route('/api/produits', methods=['POST'])
def create_produit():
    d = request.get_json()
    conn = get_db()
    with conn.cursor() as cur:
        cur.execute(
            "INSERT INTO produits (nom, prix, categorie) VALUES (%s, %s, %s)",
            (d['nom'], d['prix'], d.get('categorie', ''))
        )
        conn.commit()
        new_id = cur.lastrowid
    conn.close()
    return jsonify({"id": new_id, "message": "Produit créé"}), 201

# DELETE /api/produits/<id>
@app.route('/api/produits/<int:id>', methods=['DELETE'])
def delete_produit(id):
    conn = get_db()
    with conn.cursor() as cur:
        cur.execute("DELETE FROM produits WHERE id = %s", (id,))
        conn.commit()
    conn.close()
    return jsonify({"message": "Produit supprimé"})

if __name__ == '__main__':
    app.run(debug=True, host='0.0.0.0', port=5000)
Sécurité : toujours utiliser les %s avec les paramètres séparés — jamais concaténer directement du texte utilisateur dans une requête SQL (risque d'injection SQL).
À retenir — MySQL
  • DictCursor retourne des lignes sous forme de dictionnaires (compatible JSON)
  • Toujours appeler conn.commit() après INSERT/UPDATE/DELETE
  • Toujours fermer la connexion avec conn.close()
  • Utiliser %s pour les paramètres, jamais f-strings dans les requêtes SQL

Exercices & Travaux PratiquesSQL avancé : jointures, transactions, index, optimisation

Ces exercices couvrent MySQL des requêtes de base à la conception de schémas complexes, en passant par l'optimisation des performances et la sécurité des données.

Exercices de compréhension

📝 Exercice 1 — Requêtes SELECT avancées Facile

  1. À partir d'une table employes(id, nom, prenom, salaire, departement_id, date_embauche), écrivez les requêtes pour : la liste triée par salaire décroissant, les employés embauchés entre 2020 et 2023, le salaire moyen par département, les départements où la moyenne des salaires dépasse 3000 €.
  2. Utilisez les fonctions de fenêtrage : RANK() OVER (PARTITION BY departement_id ORDER BY salaire DESC) pour classer les employés par salaire au sein de chaque département.
  3. Trouvez les doublons dans une table (même nom + prénom) avec GROUP BY … HAVING COUNT(*) > 1.

📝 Exercice 2 — Jointures complexes Moyen

  1. Créez un schéma e-commerce : clients, commandes, lignes_commande, produits, categories.
  2. Écrivez les requêtes : top 10 clients par CA total, produits jamais commandés (LEFT JOIN + IS NULL), CA mensuel sur 12 mois (GROUP BY + YEAR/MONTH), panier moyen par client.
  3. Utilisez des Common Table Expressions (CTE) avec WITH pour décomposer une requête complexe en étapes lisibles.
  4. Comparez les performances de votre requête avec et sans index. Utilisez EXPLAIN SELECT … pour analyser le plan d'exécution.

📝 Exercice 3 — Procédures stockées & Triggers Avancé

  1. Créez une procédure stockée creer_commande(client_id, produit_id, quantite) qui vérifie le stock, insère la commande, décrémente le stock et lève une erreur SQL si le stock est insuffisant.
  2. Créez un trigger AFTER UPDATE sur la table produits qui insère dans une table audit_log l'ancienne valeur, la nouvelle valeur, l'utilisateur et la date de modification à chaque changement de prix.
  3. Créez une vue vue_commandes_details qui joint toutes les tables et présente les données formatées pour un export.

Travaux Pratiques

🔬 TP — Conception et implémentation d'une base de données de gestion scolaire Avancé — 3h

Objectif : Modéliser, créer et alimenter une base de données complète pour un établissement scolaire.

  1. MCD : Dessiner le Modèle Conceptuel de Données avec les entités : Élève, Professeur, Classe, Matière, Note, Absence, Emploi du temps.
  2. MLD : Transformer en Modèle Logique (tables SQL avec clés primaires et étrangères).
  3. Implémentation : Créer toutes les tables avec les contraintes (NOT NULL, UNIQUE, CHECK, FOREIGN KEY ON DELETE CASCADE).
  4. Données de test : Insérer 30 élèves, 10 professeurs, 5 classes, 8 matières et 200 notes aléatoires.
  5. Requêtes métier : Bulletin de notes d'un élève, classement de la classe, taux d'absence par matière, statistiques par professeur.
  6. Backup : Exporter la base avec mysqldump et documenter la procédure de restauration.
